<objective>
## Goal
Make CODI usable by default on real PALS projects without requiring symbol-rich roadmap prose or manual configuration guesswork. Phase 172 couples the shared planning workflow to CODI just enough that prose-heavy phase objectives can surface explicit CODI targets, registers CODI as a default-on `pals.json` module contract, and documents/setup-validates the safe-skip path when `pi-codegraph` or a usable codegraph index is absent. No PLAN.md template changes. No new codegraph tools.

## Purpose
Phase 171 proved the live `impact` integration works, but it also exposed the two remaining adoption gaps from the source plan's open questions: (1) prose-heavy phase descriptions make CODI skip too often because the extractor only acts on explicit names, and (2) users do not yet get a clear default-on config/documentation story for when `pi-codegraph` or `.codegraph/` support is missing. This phase closes those gaps while keeping v0.1 honest and lightweight: optional, source-grounded symbol seeding before plan construction; explicit `modules.codi` config/schema guidance; and documentation that says planning continues cleanly when codegraph is unavailable or the repo is not indexed.

## Output
- Updated `kernel/workflows/plan-phase.md` so the shared planning workflow can do a light, source-grounded CODI-target seed pass for prose-heavy objectives before pre-plan dispatch, then reuse those seeds during the normal `analyze_scope` pass without adding new PLAN template sections.
- Updated `kernel/workflows/init-project.md`, `kernel/references/pals-json-schema.md`, and repo-root `pals.json` so fresh, migrated, and dogfooded config surfaces expose default-on `modules.codi` guidance with clear safe-skip wording.
- Updated `modules/codi/references/codi.md` and `README.md` so users get concise setup UX for CODI (`pi-codegraph` optional, indexing expectations, graceful skip behavior) without implying CODI blocks planning.
- Updated `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` so the shared workflow/config/docs contract fails loudly if the new coupling/distribution markers disappear or drift into paragraph-coupled prose assertions.
</objective>
