<objective>
## Goal
Create the v2.60 Phase 257 S6 `lifecycle-ui` extraction contract and bounded Phase 258 handoff before any runtime/source extraction work begins.

## Purpose
PALS is continuing the Pi extension sibling-module extraction wave after S5 (Phase 239), S1 (Phase 243), S2 (Phase 246), S3 (Phase 250), and S7 (Phase 254) shipped. S6 owns the visible Pi lifecycle status / widget / loop badge surface and the `syncLifecycleUi` call that mutates Pi UI through `ctx`. This subsystem consumes outputs from S5 (`RecentModuleActivity`) and parsed `PalsStateSnapshot` and is the first **runtime-coupled** sibling extraction (mutates Pi UI rather than purely producing strings); it is no longer purely an internal helper. This plan turns the v2.59 closure default recommendation into an executable contract boundary while preserving lifecycle-UI rendering behavior, the no-UI-only-lifecycle-decisions boundary, S4 canonical reply delivery, `.paul/*` artifact authority, transcript-visible canonical replies, no inferred merge intent, GitHub Flow gates, parent-owned APPLY, module evidence, and command-output validation truth.

## Output
- New docs-only contract: `docs/PI-NATIVE-LIFECYCLE-UI-EXTRACTION-CONTRACT.md`.
- Updated extraction target baseline: `docs/PI-NATIVE-EXTENSION-EXTRACTION-TARGET-BASELINE.md` records the Phase 257 S6 promotion and Phase 258 handoff while preserving S1/S2/S3/S5/S7 history verbatim.
