<objective>
## Goal
Implement the next refinement of the Pi runtime event/context model so PALS activation and lifecycle-context injection feel more deliberate, more coherent, and less heuristic-driven without creating Pi-only workflow truth.

## Purpose
This phase turns the earlier Phase 56 strategy decision into implementation work. It should make Pi use the hybrid event model more faithfully by treating explicit command entry as the strongest activation signal, moving primary injection decisions toward `before_agent_start`, and reducing the architectural weight carried by the `context` hook.

## Output
- Updated Pi extension runtime logic in `drivers/pi/extensions/pals-hooks.ts`
- Updated Pi extension README describing the refined event/injection model accurately
- Expanded Pi validation coverage proving the refined activation model stays bounded, artifact-derived, and non-authoritative
- Plan summary artifact at `.paul/phases/62-runtime-context-event-refinement/62-01-SUMMARY.md`
</objective>
