<objective>
## Goal
Create `docs/PALS-RISK-TIERED-LIFECYCLE-CONTRACT.md`, a contract-first design for proportional PALS lifecycle safety tiers before any workflow behavior is relaxed or automated.

## Purpose
Phase 234 found that PALS safety is valuable but process ceremony is not yet proportional to risk. Phase 235 turns that finding into an explicit contract that future phases can implement safely: tiny/doc-only, normal, risky, and hotfix work may have different evidence expectations, but core lifecycle authority and non-bypass gates remain intact.

## Output
A single docs-first contract artifact defining:
- risk-tier criteria for tiny/doc-only, normal, risky, and hotfix work
- required evidence expectations per tier
- explicit non-bypass rules for APPLY approval, validation truth, GitHub Flow gates, module blockers, parent-owned APPLY, lifecycle writes, `.paul/*` artifact authority, command-output truth, and full authoritative read fallback
- routing/implementation boundaries for future workflow changes
- verification markers proving the contract did not change runtime or lifecycle behavior
</objective>
