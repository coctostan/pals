# Plan 240-01 — Product Happy Path + Scenario Evidence (first-run scenario doc)

<objective>
**Goal:** Create a single bounded first-run happy-path / scenario-evidence document at `docs/PALS-FIRST-RUN-HAPPY-PATH.md` that walks a new user through one tiny end-to-end PALS loop (init → plan → apply → unify) on a toy project, names exactly what PALS protected or saved at each step, and stays small enough not to become another brittle validation surface.

**Purpose:** Address Phase 234 weakness #5 ("First-run product story is dense") and the audit's recommended Phase 240 scope. Make PALS easier to evaluate by showing user value before architecture, while preserving every non-bypass rule (`.paul/*` artifact authority, parent-owned APPLY, GitHub Flow gates, validation truth, no telemetry, no hidden state, no auto-approval, full authoritative read fallback, command-output truth).

**Output:** New doc `docs/PALS-FIRST-RUN-HAPPY-PATH.md`; one short README pointer line; one bounded cross-harness TAP marker assertion that the doc exists and carries the required derived-aid authority language. Final SUMMARY at `.paul/phases/240-product-happy-path-scenario-evidence/240-01-SUMMARY.md`.
</objective>
