<objective>
## Goal
Harden the repo-source Pi guided workflow runtime so guided plan review, APPLY approval, checkpoint, resume-next, continue-to-UNIFY, phase/milestone transition, and merge-gate routing prompts can only produce explicit, transcript-visible canonical user-message replies and never become UI-only lifecycle authority.

## Purpose
v2.48 must turn the Phase 206 guided workflow evidence contract into runtime behavior before documentation and validation surfacing. This plan keeps the change bounded to `drivers/pi/extensions/pals-hooks.ts`, preserving `.paul/*` artifact truth, PLAN/APPLY/UNIFY ownership, module dispatch evidence, GitHub Flow gates, parent-owned APPLY, command-output validation truth, and full authoritative reads for consequential work.

## Output
- `drivers/pi/extensions/pals-hooks.ts` includes the contract-backed guided moment inventory, including merge-gate routing as a notify/select-only guided moment.
- Guided workflow reply delivery remains exclusively through `pi.sendUserMessage` / `deliverAs: "followUp"` after explicit confirm/select results.
- Runtime source contains durable markers for no auto-approval, no auto-continue, no skipped checkpoints, no UI-only lifecycle decisions, no inferred merge intent, notify-only no-reply behavior, and non-authority boundaries.
- Repo-source Pi extension can be installed with `PALS_ROOT="$PWD" bash drivers/pi/install.sh` and the existing Pi/cross-harness validation suites remain green.
</objective>
