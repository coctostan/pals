<objective>
## Goal
Replace the inherited CODI marker-string validation drift with focused semantic guardrails, add reusable module-instruction semantic assertion helpers, and record v2.64 closure evidence so Phase 278 validates module-instruction behavior rather than literal prose markers.

## Purpose
Phase 278 closes the v2.64 module instruction audit milestone. Phases 273-277 rewrote module instruction surfaces to be shorter, bounded, advisory/enforcement-correct, and source-grounded. The remaining known gap is validation drift: `tests/cross-harness-validation.sh` still expects several stale CODI marker strings that Phase 276 intentionally removed or tightened. This plan updates validation to reject marker-only compliance while preserving the real behavioral contracts: bounded source selectors, safe CODI skip behavior, post-unify CODI history instrumentation, visible dispatch evidence, and installed/source registry consistency.

## Output
A focused guardrail update with:
- a compact audit of the 7 inherited CODI cross-harness failures and the semantic contract each should verify;
- a reusable shell helper for semantic module-instruction assertions used by cross-harness validation;
- cross-harness tests updated from brittle marker strings to semantic checks over repo source and freshly installed Claude Code / Pi runtime surfaces;
- validation evidence showing `tests/cross-harness-validation.sh`, `tests/pi-end-to-end-validation.sh`, and `tests/helpers/artifact_consistency.sh` pass or any remaining failures are diagnosed with source evidence;
- a closure note in `docs/PALS-MODULE-INSTRUCTION-AUDIT-CONTRACT.md` documenting the Phase 278 semantic guardrail pattern without changing runtime/module behavior.
</objective>
