<objective>
## Goal
Add a bounded install-time codegraph-index recommendation to the Pi installer, align CODI's manifest/reference/README with the phase-173 value-envelope findings, and pin the success-log `R` convention without widening CODI beyond its v0.1 advisory scope.

## Purpose
Phase 173 selected three remaining loose ends after the extractor iteration: (1) default-on CODI should stay honest on repos without a local index marker, (2) users should understand where CODI actually helps, and (3) the success-log resolved-count convention must stop being ambiguous before the re-trial milestone. Phase 174 shipped the source-file selector extractor and made this planning run the first live v2.40 session that can use a read-only TS evidence file in `<context>`. Phase 175 closes the install/docs/format gaps while keeping the implementation bounded to installer, manifest/reference, README, a single changelog entry, and verification surfaces.

## Output
- Updated `drivers/pi/install.sh` so CODI-enabled installs emit a single non-blocking recommendation when no local `.codegraph/graph.db` marker is detected.
- Updated `modules/codi/module.yaml`, `modules/codi/references/codi.md`, `README.md`, and `CHANGELOG.md` so CODI's value envelope is described honestly, the docs use the bounded “can improve boundary specificity” framing on structural call-site evidence, and the success-log format explicitly defines `R` as resolved-with-call-sites only.
- Updated `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` only if they need minimal stable semantic assertions for the new installer/docs markers.
</objective>
