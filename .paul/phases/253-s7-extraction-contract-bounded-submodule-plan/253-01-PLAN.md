<objective>
## Goal
Create the v2.59 Phase 253 S7 `pals-context-injection` extraction contract and bounded Phase 254 handoff before any runtime/source extraction work begins.

## Purpose
PALS is continuing the Pi extension sibling-module extraction wave after S5 (Phase 239), S1 (Phase 243), S2 (Phase 246), and S3 (Phase 250) shipped. S7 owns the PALS context payload that is injected before agent start (`PRIMARY_INJECTION_EVENT`) and the supporting `context` event, plus the message-stream housekeeping that keeps only the latest PALS context message. This subsystem owns the explicit `[PALS_STATE_AUTHORITY=.paul/STATE.md]` and `[PALS_ACTIVATION_SIGNAL]` tags that consumers — including `/paul-resume` itself — rely on. This plan turns the v2.58 closure default recommendation into an executable contract boundary while preserving `.paul/*` authority, exact authority/activation tags, the legacy-header migration path, the custom-message-type identity, transcript-visible canonical replies, no inferred merge intent, and command-output validation truth.

## Output
- New docs-only contract: `docs/PI-NATIVE-PALS-CONTEXT-INJECTION-EXTRACTION-CONTRACT.md`.
- Updated extraction target baseline: `docs/PI-NATIVE-EXTENSION-EXTRACTION-TARGET-BASELINE.md` records the Phase 253 S7 promotion and Phase 254 handoff while preserving S1/S2/S3/S5 history verbatim.
- Updated modularization contract: `docs/PI-NATIVE-EXTENSION-MODULARIZATION-CONTRACT.md` adds a Phase 253 outcome/handoff note pointing S7 extraction work at the new contract while preserving S4/S6/S8 deferrals.
