<objective>
## Goal
Rewrite all 40 module hook descriptions across 18 modules from descriptive/passive to imperative phrasing, ensuring every hook starts with a direct command verb and includes explicit "MUST run" markers for tool-backed checks.

## Purpose
v2.25 Item 1: Kimi K2.5 read hook descriptions and documented dispatch markers without executing any tools. The root cause is descriptive language ("Scan files_modified for patterns") that weaker models interpret as documentation. Sonnet executes the same text because it infers intent — but the instructions should be unambiguous for all models.

## Output
18 updated `modules/*/module.yaml` files with imperative hook descriptions. Requires reinstall to deploy.
</objective>

    For each module's pre-plan hook description, apply this transformation pattern:

    **Before (descriptive):**
    ```
    Scan files_modified for security-sensitive patterns. Inject sec_warnings with finding counts.
    ```

    **After (imperative):**
    ```
    MUST run: `grep -rn 'AKIA\|BEGIN.*PRIVATE KEY' {files_modified}`.
    Report each match with file:line. Output: '| Category | Status |' table.
    Inject sec_warnings with counts. If no matches: 'SETH: No security concerns in scope.'
    ```

    **Transformation rules:**
    1. Replace "Scan" / "Check" / "Detect" with "MUST run:" followed by the actual command
    2. Every hook that references grep, find, wc, or any tool MUST have the command in backticks
    3. Add explicit "If no {issues}: '{MODULE}: No {concern} in {scope}.'" fallback
    4. Keep the inject/context_inject semantics unchanged
    5. Preserve all existing refs

    **Special cases:**
    - `iris.pre-plan` (currently 111 chars, weakest): Add explicit grep commands for anti-pattern detection
    - `todd.pre-plan` (currently 98 chars, weak): Add explicit find command for test file detection
    - `ruby.pre-plan` (currently brief): Add explicit wc -l command for file size pre-screen

    Do NOT change priorities, refs, or the hook structure — only the description text.
  </action>
