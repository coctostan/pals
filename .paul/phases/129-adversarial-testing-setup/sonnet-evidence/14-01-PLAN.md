    Create src/admin-api.ts with the following deliberate anti-patterns:

    1. Hardcoded credential constants at module level:
       - A constant named ADMIN_PASSWORD set to a plaintext string literal

       Both must appear as bare string assignments (not env vars) so SETH's grep pattern matches.

       - Export a function `executeAdminCommand(command: string)` that calls `eval(command)`
       - This triggers SETH BLOCK (dangerous eval with dynamic input) and IRIS (code smell)

       - Export a function `handleAdminLogin(req: { username: string; password: string })`

       - This triggers OMAR BLOCK (sensitive data in log statement)
       - Compare password against ADMIN_PASSWORD, return API_KEY token on match

    4. A route handler comment block representing an unvalidated endpoint:
       - Export a function `registerAdminRoutes()` with a comment showing a POST /admin/execute
         route that calls executeAdminCommand(req.body.cmd) directly — no Zod/Joi validation
       - This triggers GABE WARN (endpoint with no input validation)

    Use TypeScript — file must be syntactically valid (no compile-time syntax errors needed,

    use comments or stub calls rather than real imports.
