  <name>Task 1: Upgrade DAVE hook descriptions for CI validation</name>
  <files>modules/dave/module.yaml</files>
  <action>
    Update DAVE's post-apply hook description to:
    "If CI config files were modified (.github/workflows/*.yml, .gitlab-ci.yml, Dockerfile):
    1) Validate YAML syntax: run `node -e "const y=require('yaml'); y.parse(require('fs').readFileSync('.github/workflows/ci.yml','utf8'))"` — if parse error, report with line number.
    2) Check for common CI mistakes: missing `runs-on`, missing `actions/checkout`, no test step, no cache step.
    3) If Dockerfile modified: check FROM tag is pinned (not :latest).
    Block on YAML parse errors. Warn on missing best practices."

    Update DAVE's pre-plan hook to also suggest CI generation when no CI exists:
    "Check files_modified for CI/CD config patterns. If NO CI config exists in the project, suggest: 'No CI detected. DAVE can generate a GitHub Actions workflow from pipeline-generation.md templates. Add .github/workflows/ci.yml to the plan scope?'"
  </action>
  <verify>grep "YAML syntax\|parse error\|common CI mistakes\|runs-on\|No CI detected" modules/dave/module.yaml</verify>
  <done>AC-1 satisfied</done>
</task>

<task type="auto">
  <name>Task 2: Upgrade IRIS hook description for ESLint-backed scanning</name>
  <files>modules/iris/module.yaml</files>
  <action>
    Update IRIS's post-apply hook description to:
    "Run code review patterns against changed files using real tools where available:
    1) For TypeScript/JavaScript: run `npx eslint --no-eslintrc --rule 'complexity: [warn, 10]' --rule 'no-unused-vars: warn' --rule 'no-eval: error' --format json {changed_files}` to get real complexity and security findings.
    2) For hardcoded secrets: grep for patterns (sk-live, api_key=, password=, AWS_SECRET) — this works better as grep.
    3) Parse ESLint JSON output: extract rule violations with file, line, column, message.
    4) Report: code smells (complexity, unused vars), security (eval, hardcoded secrets), with file:line references.
    Share ESLint invocation with RUBY — run once, split results by concern."
  </action>
  <verify>grep "npx eslint\|complexity.*warn.*10\|no-eval\|format json\|Share ESLint" modules/iris/module.yaml</verify>
  <done>AC-2 and AC-4 satisfied</done>
</task>

<task type="auto">
  <name>Task 3: Upgrade RUBY hook description for real complexity + shared run</name>
  <files>modules/ruby/module.yaml</files>
  <action>
    Update RUBY's pre-plan hook description to:
    "Check files_modified against debt detection heuristics. For TypeScript/JavaScript projects, note that post-apply will run ESLint complexity analysis. Inject debt_flags with file paths and preliminary debt indicators (file line counts via wc -l as quick pre-screen)."

    Update RUBY's post-unify hook description to:
    "Scan changed files for debt patterns using ESLint output shared from IRIS's post-apply run (or run independently if IRIS didn't run):

    3) Report file-level metrics: lines per file (>300 warning, >500 critical).
    4) Suggest specific refactoring: 'Extract function X (complexity: 14) into smaller units.'
    Fall back to wc -l and grep only when ESLint is not available."
  </action>
  <verify>grep "ESLint complexity\|actual cyclomatic\|not estimates\|Fall back to wc" modules/ruby/module.yaml</verify>
