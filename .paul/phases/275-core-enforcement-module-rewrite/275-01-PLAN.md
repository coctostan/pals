<objective>
## Goal
Audit and rewrite the first core enforcement module cohort — TODD, WALT, and DEAN manifest hook descriptions plus their primary enforcement references — so blocking behavior is concrete, evidence-based, recoverable, and aligned with the Phase 273/274 module dispatch contract.

## Purpose
Phase 275 turns the v2.64 module instruction audit contract toward modules that can block PLAN/APPLY progress. These instructions are currently powerful but too often rely on broad shell commands, marker phrases, vague pass/block claims, or implicit lifecycle authority. The first Phase 275 plan should address the highest-risk enforcement surfaces before later plans handle overlays, secondary references, or domain modules.

## Output
A bounded core-enforcement rewrite with:
- compact audit of TODD, WALT, and DEAN approved files and proposed file/section order;
- blocking user approval checkpoint before source edits;
- approved section-by-section collaboration loop for every mutated file/section: current excerpt or bounded summary, concrete problem, proposed wording/focused diff, affected enforcement semantics, verification expectation, user approval or wording change, then edit only that approved section;
- re-read verification after each approved section edit before moving to the next section;
- final validation evidence showing enforcement modules block only with concrete evidence, name recovery handling, preserve workflow-owned lifecycle decisions, and use hook-local refs.
</objective>
