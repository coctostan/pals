## Goal
Deep-dive how 10+ agentic coding frameworks approach planning — decomposition, spec generation, context gathering, scope management, plan structure. Produce a findings document with adopt/adapt/reject decisions, then implement concrete improvements to PALS plan-phase.md.

Planning is where output quality is won or lost. Bad plans = bad code. PALS plan-phase.md is functional but was designed from first principles — we haven't systematically learned from what others do well. This phase fixes that.

## Output
1. `planning-research.md` — deep competitive findings with specific techniques cited
2. Modified `kernel/workflows/plan-phase.md` — concrete improvements adopted from research

    **Batch 4 (parallel — additional):**
    - Cursor rules / .cursorrules: how rules shape planning
    - MetaGPT: multi-agent planning with SOP (Standard Operating Procedure)
    - GPT Pilot: step-by-step planning with human checkpoints

    For each framework, research and document:
    1. **Decomposition strategy**: How does it break work into steps?
    2. **Context gathering**: How does it decide what files/info to read?
    3. **Scope management**: How does it prevent scope creep?
    4. **Plan structure**: What format does the plan take?
    5. **Verification approach**: How does it know the plan is good?
    6. **Error handling**: What happens when a plan step fails?
    7. **Unique technique**: What's the one thing this framework does that nobody else does?

    Compile into planning-research.md with per-framework sections and a cross-cutting comparison matrix.
