<objective>
## Goal
Add pals.json migration so existing projects automatically get new fields, modules, and settings when PALS is updated. Add a schema reference so the expected shape is documented in one place.

## Purpose
A v0.5 project has 3 modules and 6 git fields. Current PALS expects 18 modules, 14 git fields, planning config, and preferences. There's no migration path — old projects silently run with incomplete config. The `/paul:config` command shows only 8 modules hardcoded. This needs to work automatically.

## Output
- pals-json-schema.md reference defining the canonical pals.json shape
- init-project.md detects existing pals.json and migrates missing fields
- config.md reads modules from the installed modules.yaml instead of a hardcoded list
- `.paul/phases/128-install-config-init-audit/128-02-SUMMARY.md`
</objective>

<task type="auto">
  <name>Task 2: Add migration step to init-project.md</name>
  <files>kernel/workflows/init-project.md</files>
  <action>
    Add a `migrate_pals_json` step early in init-project.md (after detecting an existing project). This step:

    1. Reads existing pals.json
    2. Compares against the canonical schema (from Task 1)
    3. For each missing field, adds it with its default value
    4. For `modules` section: adds any modules present in the installed `modules.yaml` but missing from pals.json, with `enabled: true`
    5. Does NOT overwrite any existing user values
    6. Shows a summary of what was added:
       ```
       pals.json migration:
         Added 10 modules: arch, seth, pete, gabe, luke, aria, dana, omar, reed, vera
         Added git.workflow: "none" (default)
         Added git.base_branch: "main" (default)
         Added planning.default_collaboration: "medium" (default)
         (3 fields unchanged, 8 modules unchanged)
       ```
    7. Writes the updated pals.json

    This should also run when init detects `.paul/` already exists (brownfield re-init / upgrade path).
  </action>
  <verify>
    ```bash
    grep "migrate_pals_json\|migration" kernel/workflows/init-project.md | head -5
    ```
  </verify>
  <done>AC-2 satisfied.</done>
</task>
