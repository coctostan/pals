<objective>
## Goal
Create the contract-first foundation for v2.71 HTML presentation packets: an authoritative contract document that fixes packet authority, storage, packet types, audience modes, static HTML rules, source inputs, and non-goals, plus a reusable template model for derived static HTML briefs.

## Purpose
v2.71 adds human-review briefs without reviving live UI dependencies or weakening `.paul/*` lifecycle truth. Phase 299 establishes the boundary and template vocabulary before later phases generate or surface packet instances for milestone, PLAN, APPLY, and UNIFY moments. Contract-first keeps later workflow work narrow, reviewable, and aligned with the project rule that runtime assistance is derived, bounded, source-cited, and disposable.

## Output
- `docs/PALS-HTML-PRESENTATION-PACKETS-CONTRACT.md` — new authoritative contract for v2.71 packet behavior and boundaries.
- `kernel/templates/HTML-PRESENTATION-PACKET.md` — new template-model document containing the static HTML packet skeleton and field conventions.
- No workflow wiring, no packet generation, no validation harness changes; those are Phases 300-302.
</objective>
