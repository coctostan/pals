<objective>
## Goal
Add quality trend visibility to PALS via a `/paul:quality` command that reads QUALITY-HISTORY.md and presents trend analysis showing whether project quality is improving, stable, or degrading over time.

## Purpose
WALT gates individual plans via pre/post-apply hooks and records history in QUALITY-HISTORY.md via post-unify, but there's no way to *view* that accumulated data. This gives users a dashboard over their quality trajectory with trend detection and actionable insights.

## Output
- `modules/walt/references/quality-trends.md` — trend analysis logic (pattern detection, streak analysis)
- `kernel/commands/paul/quality.md` — the `/paul:quality` command
- Updated `modules/walt/module.yaml` — new reference registered
</objective>

<acceptance_criteria>

## AC-1: Quality Trend Dashboard
```gherkin
Given a project with .paul/QUALITY-HISTORY.md containing plan entries
When /paul:quality is invoked
Then a dashboard is displayed showing:
  - Cumulative trajectory for each metric (Tests, Coverage, Lint, Types)
  - Recent plan history (last 5-10 entries)
  - Overall trend verdict (improving/stable/degrading)
```

## AC-2: Trend Pattern Detection
```gherkin
Given quality history with 3+ entries
When trends are analyzed
Then patterns are detected:
  - Streaks (consecutive improvements or degradations)
  - Regressions (metric that was improving then reversed)
  - Stalls (metric stuck at same value for 3+ plans)
```

## AC-3: No History Graceful Handling
```gherkin
Given a project with no .paul/QUALITY-HISTORY.md
When /paul:quality is invoked
Then it displays a helpful message explaining how quality tracking starts
And suggests running a plan with WALT enabled
```

    Create a reference document for trend analysis logic.

    Structure using XML sections:
    1. Overview — philosophy of trend tracking (trajectory tells the project's health story)
    2. Trend detection patterns:
       - **Streak detection:** 3+ consecutive same-direction movements (e.g., Tests: ▲▲▲ = "3-plan improvement streak")
       - **Regression detection:** metric that was improving for 2+ plans then reversed (e.g., Lint: ▲▲▼ = "regression after improvement")
       - **Stall detection:** metric unchanged for 3+ plans (e.g., Coverage: ●●● = "stalled at 82%")
       - **Baseline only:** single entry, no trend possible
    3. Overall project health verdict:
       - Healthy: majority ▲ or ●, no active regressions
       - Attention needed: any ▼ or active regression
       - Degrading: majority ▼, multiple regressions
       - Insufficient data: fewer than 3 entries
    4. Presentation format — how to display trends in the dashboard

    Follow quality-delta.md's style (trajectory indicators, edge cases).
    Keep concise — this is guidance for the command, not a workflow.

<task type="auto">
  <name>Task 2: Create /paul:quality command</name>
  <files>kernel/commands/paul/quality.md</files>
  <action>
    Create the command file following coverage.md's pattern (frontmatter + process steps).

    Frontmatter:
    - name: pals:quality
    - description: Display quality trend dashboard from WALT history
    - allowed-tools: [Read, Bash, Glob, Grep]

    Command workflow:
    1. Check for .paul/QUALITY-HISTORY.md
       - If not found: display helpful message (AC-3)
       - If found: read and parse

    2. Parse history:
       - Read cumulative trajectory lines (Tests, Coverage, Lint, Types)
       - Read plan history table rows
       - Count total entries

    3. Analyze trends (per quality-trends.md):
       - For each metric: detect streaks, regressions, stalls
       - Compute overall project health verdict

    4. Present dashboard:
       ```
       ════════════════════════════════════════
       WALT QUALITY TRENDS
       ════════════════════════════════════════

       ── Trajectory ─────────────────────────
       Tests:    0→15→17→21        ▲ improving (4-plan streak)
       Coverage: —→78%→82%→82%     ● stalled at 82%
       Lint:     12→10→5→0         ▲ improving (clean!)
       Types:    0→0→0→0           ● stable

       ── Recent History ─────────────────────
       | Plan  | Tests | Coverage | Lint | Verdict |
       |-------|-------|----------|------|---------|
       | 13-01 | 21    | 82%      | 0    | ▲       |
       | 12-01 | 17    | 82%      | 5    | ▲       |
       | 11-01 | 15    | 78%      | 10   | ▲       |

       ── Health ─────────────────────────────
       Overall: Healthy
       Streaks: Tests improving (4 plans), Lint improving (4 plans)
       Attention: Coverage stalled at 82% for 2 plans
       ════════════════════════════════════════
       ```

    5. Suggest actions based on health:
       - If regressions: "Recent regression in [metric] — investigate last plan"
       - If stalls: "Coverage stalled — run /paul:coverage to find gaps"
       - If healthy: "Quality trajectory looks good!"
       - If no data: "Run a plan with WALT enabled to start tracking"

    Error handling:
    - No .paul/ directory → "No PAUL project. Run /paul:init first."
    - Empty history → "No quality data yet."
    - Single entry → "Only baseline — need 2+ entries for trends"
  </action>
