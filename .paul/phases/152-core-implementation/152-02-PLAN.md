<task type="auto">
  <name>Task 1: Create /paul:review Skill</name>
  <files>drivers/pi/skills/paul-review/SKILL.md</files>
  <action>
    Create `drivers/pi/skills/paul-review/SKILL.md` following the design from rev-design.md Sections 5 and 6.

    **Frontmatter:**
    ```yaml
    ---
    name: paul-review
    description: "Run thorough code review via REV subagent — spawns an isolated reviewer with fresh context for unbiased, senior-dev-level review across 8 dimensions."
    ---
    ```

    **Structure:**
    1. `# PAUL Review` — heading
    2. `## What This Does` — explain: dispatches code-reviewer subagent with assembled diff + context for thorough review
    3. `## How to Execute` — numbered steps:
       a. Read `.paul/STATE.md` and `pals.json` (check modules.rev config)
       b. Read the installed module refs: `../modules/rev/references/review-prompt.md` and `review-checklist.md`
       c. Present scope selection menu (4 options from design Section 6):
          - [1] Current plan's changes (diff since branch start)
          - [2] Specific files (user picks)
          - [3] Full branch vs base_branch
          - [4] Uncommitted changes
       d. Assemble context: run diff command, read changed files, read AGENTS.md, find test files
       e. Check for pi-subagents dependency (Agent tool available). If not:
          - Warn and offer [1] Skip or [2] In-session fallback
       f. Build prompt from review-prompt.md template, substituting placeholders
       g. Dispatch Agent() with subagent_type "code-reviewer", foreground mode
       h. If pals.json modules.rev.model is set, pass as model override
       i. Display results with verdict (READY / NOT READY / READY WITH CONCERNS)
    4. `## Key Behavior` — bullet points:
       - On-demand only — not a lifecycle hook
       - Foreground dispatch (blocks until complete)
       - Configurable model via pals.json
       - Graceful degradation if pi-subagents missing
    5. `## Output` — reviewer findings displayed inline

    Keep under 80 lines total. Be specific enough to execute but not overly verbose.
  </action>
  <verify>
    - ls drivers/pi/skills/paul-review/SKILL.md
    - grep 'paul-review' drivers/pi/skills/paul-review/SKILL.md
    - grep 'code-reviewer' drivers/pi/skills/paul-review/SKILL.md
    - grep 'review-prompt.md' drivers/pi/skills/paul-review/SKILL.md
    - wc -l drivers/pi/skills/paul-review/SKILL.md (under 80)
  </verify>
  <done>AC-1 satisfied</done>

<task type="auto">
  <name>Task 2: Add Merge Gate Review Step + Pi Extension Command + Reinstall</name>
  <files>kernel/workflows/unify-phase.md, drivers/pi/extensions/pals-hooks.ts</files>
  <action>
    **Add merge gate review step to unify-phase.md:**

    Insert a new step `merge_gate_review` between `merge_gate_pr` and `merge_gate_merge` with:
    - Priority: `after-merge-gate-pr`
    - Condition: `rev_pr_review_enabled`
    - Check `pals.json modules.rev.pr_review` — if false or REV disabled, skip silently
    - If true: assemble full-branch diff review prompt, dispatch Agent() with code-reviewer
    - Parse verdict. If CRITICAL findings AND `pr_review_block_on_critical=true`: BLOCK merge
    - If IMPORTANT/MINOR only: display as advisory, proceed
    - If READY: display "Code review: ✓ READY" and proceed

    Update the merge gate display in `merge_gate_cleanup` step to show 7 steps:
    ```
    1. PR exists:     ✓
    2. CI passing:    ✓ / skipped
    3. Code review:   ✓ READY / ⚠️ CONCERNS / ⛔ BLOCKED / skipped
    4. Reviews:       ✓ / skipped
    5. PR merged:     ✓
    6. Base synced:   ✓
    7. Branch clean:  ✓
    ```

    **Add paul-review command to pals-hooks.ts:**

    1. Add to the COMMANDS array (after paul-help entry):
    ```typescript
    {
      name: "paul-review",
      description: "Run thorough code review via REV subagent",
      skill: "paul-review",
      guidance: "Pi convenience wrapper → canonical /skill:paul-review",
    },
    ```

    2. Update the `detectCommandSignal` regex to include `review`:
    Change: `paul-(?:init|plan|apply|unify|resume|status|fix|pause|milestone|discuss|help)`
    To: `paul-(?:init|plan|apply|unify|resume|status|fix|pause|milestone|discuss|help|review)`

    **Reinstall:**
    Run `PALS_ROOT=$(pwd) bash drivers/pi/install.sh`

    **Verify deployment:**
    - Check paul-review skill exists in installed location
    - Check pals-hooks.ts deployed with review command
  </action>
  <verify>
    - grep 'merge_gate_review' kernel/workflows/unify-phase.md
    - grep 'pr_review' kernel/workflows/unify-phase.md
    - grep 'Code review:' kernel/workflows/unify-phase.md
    - grep 'paul-review' drivers/pi/extensions/pals-hooks.ts
    - grep 'review' drivers/pi/extensions/pals-hooks.ts | grep detectCommandSignal
    - ls ~/.pi/agent/skills/pals/skills/paul-review/SKILL.md
    - grep 'paul-review' ~/.pi/agent/extensions/pals-hooks.ts
  </verify>
  <done>AC-2, AC-3 satisfied</done>
