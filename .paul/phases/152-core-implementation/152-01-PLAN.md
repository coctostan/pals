  <name>Task 1: Create REV Module</name>
  <files>modules/rev/module.yaml, modules/rev/references/review-prompt.md, modules/rev/references/review-checklist.md</files>
  <action>
    Create `modules/rev/` directory with 3 files:

    **modules/rev/module.yaml:**
    Follow the design from rev-design.md Section 2. Key points:
    - name: rev, display_name: REV
    - description: Review Engine & Verification — thorough code review via subagent dispatch
    - version: 1.0.0
    - dependencies: kernel >=2.0.0, external pi-subagents (required: false)
    - hooks: on-demand only (priority 100). Description explains this is triggered by /paul:review, not a lifecycle hook. References review-prompt.md and review-checklist.md.
    - commands: review
    - files.references: review-prompt.md, review-checklist.md

    **modules/rev/references/review-prompt.md:**
    The full review prompt template from rev-design.md Section 1. Include:
    - Context assembly instructions (diff, changed files, AGENTS.md, tests)
    - Scope-to-diff-command mapping table
    - The complete prompt template with placeholders
    - Output format specification (Summary, Findings by severity, Strengths, Verdict)

    **modules/rev/references/review-checklist.md:**
    The 8 review dimensions from rev-design.md Section 1. For each:
    - Dimension name and what to check
    - Severity guide (when Critical vs Important vs Minor)
    - 2-3 concrete examples of findings in that dimension
    Keep under 150 lines.
  </action>
  <verify>
    - ls modules/rev/module.yaml modules/rev/references/review-prompt.md modules/rev/references/review-checklist.md
    - grep 'on-demand' modules/rev/module.yaml

  </verify>
