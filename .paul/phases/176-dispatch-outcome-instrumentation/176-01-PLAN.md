<objective>

## Goal
Add a bounded CODI post-unify hook that appends exactly one row per UNIFY to a new `.paul/CODI-HISTORY.md` tally file, capturing the dispatch outcome category (6 tokens including a `no-dispatch-found` fallback), the R/U/K counts from the success log, the symbols probed, and whether a blast_radius was injected. Seed the tally with a single Phase 175 row so the file is non-empty on first write, and document the contract in a new focused reference doc `modules/codi/references/codi-instrumentation.md` (split from `codi.md` to keep both files within the 150-line module-ref cap) with bounded semantic validation coverage.
## Purpose
Phase 173's trial exposed that CODI dispatch evidence only lives in per-phase PLAN.md / SUMMARY.md artifacts today. Phase 177's re-trial rubric (`S_pass_nonnull ≥ 3`) requires reading multiple live sessions and comparing dispatch outcomes at a glance. A tally file mirroring `.paul/QUALITY-HISTORY.md`'s shape — written automatically by post-unify, one row per plan — turns that measurement from manual reconstruction into a direct read. Advisory-only: the hook records what already happened during pre-plan dispatch; it never mutates plan or module state. The hook's data source is the PLAN.md `<module_dispatch>` block (written at plan time, never mutated), with SUMMARY.md as a historical fallback, because at post-unify dispatch time the SUMMARY.md draft has not yet been finalized with carry-forward module reports (`finalize_summary` runs *after* `post_unify_hooks`).
## Output
- `modules/codi/module.yaml` gains a `post-unify` hook at priority 220 with MUST-wording that reads the plan's `<module_dispatch>` block first (primary), falls back to the SUMMARY.md Module Execution Reports second, categorizes the outcome, and appends one row to `.paul/CODI-HISTORY.md`. Hotfix mode is explicitly handled (no PLAN.md sibling → `no-dispatch-found`).
- `modules/codi/references/codi-instrumentation.md` is created as a focused reference covering the row schema, outcome taxonomy, data-source read order, plan-path resolution rules, hotfix behavior, and a worked example row. Kept within the 150-line module-ref cap.
- `modules/codi/references/codi.md` gains a short cross-reference pointer to the new instrumentation ref (no other edits), staying well within its 150-line cap.
- `.paul/CODI-HISTORY.md` is created with the header row and a single seeded `175-01` row citing `173-01-TRIAL-DATA.md` as the historical source. Symbol order preserved from the 175-01-PLAN.md Blast Radius section verbatim.
- `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` gain minimal semantic assertions for: (a) the new manifest markers, (b) the new reference doc's existence and key markers, (c) the 5 pre-plan skip-log strings present verbatim in the CODI manifest description (drift guard between pre-plan logs and the post-unify parser).
- Installed surface is refreshed so Phase 176's own UNIFY (and every subsequent UNIFY) fires the new hook end-to-end.
