<task type="auto">
  <name>Task 2: Create agent-dispatch reference document</name>
  <files>~/.pals/references/agent-dispatch.md</files>
  <action>
    Create a new reference file at ~/.pals/references/agent-dispatch.md (under 150 lines per context budget).

    Sections:

    1. **Agent Roles** — table of roles, default models, and when to use each.

    2. **Prompt Composition** — template for composing agent prompts:
       ```
       You are a {role} agent. Execute exactly as described.

       ## Rules
       {CARL global rules — always_on domain, injected by orchestrator}

       ## Task: {task.name}
       ## Files: {task.files}
       ## Action:
       {task.action}
       ## Boundaries:
       {plan.boundaries}
       ## Verify:
       {task.verify}
       ## Done:
       {task.done}
       ```
       Note: For sequential tasks where prior task output matters, add a one-line
       `## Prior: {file} was created/modified by previous task` section.

    3. **Structured Response Format** — agents MUST end their work with:
       ```
       ## Agent Result
       Status: PASS | PASS_WITH_CONCERNS | BLOCKED
       Files modified: [list]
       Approach: [1-2 sentences on what was done]
       Deviations: [none, or description of departures from plan]
       Concerns: [none, or issues worth noting]
       Verify output: [command output or check result]
       ```

    4. **Dispatch Rules:**
       - Writing agents (coder, tester) execute sequentially — never parallel. Prevents file conflicts.
       - Research agents can run in parallel (read-only).
       - Reviewer agents can run in parallel (read-only analysis).
       - CARL global rules (always_on domain) are injected into every agent prompt.
       - Domain-specific CARL rules are NOT injected (conversation-scoped).

    5. **Config Reading:**
       ```bash
       MODEL=$(jq -r '.agents.coder.model // empty' pals.json 2>/dev/null)
       ```
       If empty/missing: execute task inline (no agent dispatch).

    6. **Orchestrator Responsibilities:**
       - Compose prompt from task definition
       - Dispatch Agent tool with configured model
       - Collect structured response
       - Trust PASS results; spot-check PASS_WITH_CONCERNS via `git diff`
       - On BLOCKED: handle via bounded retry (max 2), then escalate
       - Run post-task hooks after agent returns (hooks can read modified files)

    Keep under 130 lines.
  </action>
  <verify>File exists at ~/.pals/references/agent-dispatch.md, is under 150 lines, covers all 6 sections</verify>
