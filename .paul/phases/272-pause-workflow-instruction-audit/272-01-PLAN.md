<objective>
## Goal
Rewrite the pause workflow so session handoff creation is compact, self-contained, stale-context resistant, and semantically validated without causing future `/paul:resume` sessions to over-read or prefer obsolete artifacts.

## Purpose
Phase 272 closes v2.63 by applying the instruction-audit pattern to `pause-work.md`. The current workflow preserves useful handoff semantics and GitHub Flow continuity, but it still has broad STATE/PROJECT reads, a large generic handoff template, user-question ceremony that can duplicate conversation history, and optional WIP commit wording that can leave active handoffs stale or over-detailed. The pause workflow should write enough context for a fresh session while keeping hot artifacts bounded and resume routing authoritative.

## Output
- `kernel/workflows/pause-work.md` rewritten for bounded position detection, compact context gathering, concise self-contained handoff payloads, stale handoff hygiene, GitHub Flow WIP continuity, and short resume instructions.
- Semantic guardrails in `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` requiring operational pause-workflow behavior instead of marker prose.
- Phase 272 summary after APPLY at `.paul/phases/272-pause-workflow-instruction-audit/272-01-SUMMARY.md`.
</objective>
