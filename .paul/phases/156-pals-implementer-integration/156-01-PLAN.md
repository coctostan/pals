<objective>
## Goal
Implement the first repo-controlled PALS-native implementer path for Pi so APPLY can delegate eligible bounded tasks through a repo-local `pals-implementer` agent while preserving parent-owned verification, module enforcement, fallback, and `.paul/*` lifecycle authority.

## Purpose
Phase 155 settled the contract: the design target is not the generic runtime `implementer`, but a PALS-specific, parent-controlled path using a repo-local `.pi/agents/*.md` agent definition plus explicit `Agent()` dispatch. This phase must convert that contract into the smallest additive implementation slice that makes delegated APPLY real without weakening inline APPLY semantics, REV behavior, or artifact-first lifecycle truth.

## Output
A concrete implementation slice across the current repo:
- `.pi/agents/pals-implementer.md` — stable repo-local custom agent definition for bounded APPLY work
- `kernel/workflows/apply-phase.md` — parent-controlled delegation rules, eligibility, reporting, and inline fallback
- `drivers/pi/skills/paul-apply/SKILL.md` — Pi skill guidance aligned to the delegated APPLY path
- `pals.json` — minimal repo-controlled implementer configuration surface if needed for enable/model policy
- `drivers/pi/extensions/pals-hooks.ts` — only the bounded Pi-side support needed to keep command guidance and lifecycle UX coherent with delegated APPLY
</objective>
