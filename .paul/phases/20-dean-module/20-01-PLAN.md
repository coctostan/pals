<objective>
## Goal
Create the DEAN (Dependency Evaluation & Audit Notifier) module as a new first-party pal. DEAN watches dependencies for vulnerabilities, outdated packages, and license issues, providing visibility via a `/paul:deps` command.

## Purpose
PALS has four pals (CARL, TODD, WALT) but none focused on dependency health. Projects accumulate outdated/vulnerable dependencies silently. DEAN makes dependency health visible — the same way TODD made testing visible and WALT made quality visible. This also proves the module system scales to new pals beyond the original three.

## Output
- `modules/dean/module.yaml` — module manifest (no hooks initially — read-only visibility first)
- `modules/dean/references/dependency-detection.md` — package manager detection logic (npm, pip, go, cargo, etc.)
- `modules/dean/references/vulnerability-audit.md` — audit command mapping and output parsing per ecosystem
- `modules/dean/references/dependency-principles.md` — philosophy and approach
- `kernel/commands/paul/deps.md` — the `/paul:deps` command (dashboard for dependency health)
</objective>

    Structure with XML sections covering:
    - Detection table: file indicator → package manager → ecosystem
      | Indicator | Package Manager | Ecosystem |
      | package.json | npm/yarn/pnpm | Node.js |
      | package-lock.json | npm | Node.js |
      | yarn.lock | yarn | Node.js |
      | pnpm-lock.yaml | pnpm | Node.js |
      | requirements.txt | pip | Python |
      | pyproject.toml | pip/poetry | Python |
      | Pipfile | pipenv | Python |
      | go.mod | go | Go |
      | Cargo.toml | cargo | Rust |
      | Gemfile | bundler | Ruby |
      | mix.exs | mix | Elixir |
      | composer.json | composer | PHP |
      | build.gradle / pom.xml | gradle/maven | Java |
    - Lock file detection (prefer lock file for accuracy)
    - Monorepo handling (multiple package managers in subdirectories)
    - Priority: if multiple detected, report all

    **4. Create vulnerability-audit.md** (audit command mapping):
    Structure with XML sections covering:
    - Audit commands per ecosystem:
      | Ecosystem | Audit Command | Outdated Command |
      | Node.js (npm) | npm audit --json | npm outdated --json |
      | Node.js (yarn) | yarn audit --json | yarn outdated --json |
      | Node.js (pnpm) | pnpm audit --json | pnpm outdated --json |
      | Python (pip) | pip-audit --format=json | pip list --outdated --format=json |
      | Python (poetry) | poetry audit (if available) | poetry show --outdated |
      | Go | govulncheck ./... | go list -u -m all |
      | Rust | cargo audit --json | cargo outdated |
      | Ruby | bundle audit | bundle outdated |
      | Elixir | mix deps.audit | mix hex.outdated |
      | PHP | composer audit --format=json | composer outdated --format=json |
    - Output parsing guidance per ecosystem (key fields to extract)
    - Severity classification: critical, high, medium, low
    - Error handling: tool not installed, no lock file, network errors
    - Edge cases: private registries, offline mode, workspace/monorepo

<task type="auto">
  <name>Task 2: Create /paul:deps command</name>
  <files>kernel/commands/paul/deps.md</files>
  <action>
    Create the command file following coverage.md and quality.md patterns (frontmatter + process steps).

    Frontmatter:
    - name: pals:deps
    - description: Run dependency health audit and display vulnerability/outdated dashboard
    - argument-hint: "[optional: directory path to scope analysis]"
    - allowed-tools: [Read, Bash, Glob, Grep]

    Command workflow:

    1. **Detect package manager** (per dependency-detection.md):
       - Scan project root for indicator files
       - If $ARGUMENTS provided, scan that directory instead
       - If multiple ecosystems detected, audit all
       - If none detected: display helpful message listing what DEAN looks for (AC-4)

    2. **Run audit commands** (per vulnerability-audit.md):
       - Display detected ecosystem and command being run
       - Run audit command via Bash
       - Run outdated command via Bash
       - Handle errors: tool not installed (suggest install), no lock file, network issues

    3. **Parse results**:
       - Extract vulnerability counts by severity (critical/high/medium/low)
       - Extract outdated package counts by type (major/minor/patch)
       - Calculate total dependency count from lock file or package list

    4. **Present dashboard**:
       ```
       ════════════════════════════════════════
       DEAN DEPENDENCY HEALTH
       ════════════════════════════════════════

       Ecosystem: {name} ({package_manager})
       Dependencies: {total_count}
       {scope indicator if $ARGUMENTS provided}

       ── Vulnerabilities ────────────────────
       Critical: {N}  High: {N}  Medium: {N}  Low: {N}
       {or "No known vulnerabilities ✓"}

       ── Outdated Packages ──────────────────
       Major: {N}  Minor: {N}  Patch: {N}
       {list top 5 most outdated by severity}
       {or "All packages up to date ✓"}

       ── Health ─────────────────────────────
       Overall: {Healthy / Attention needed / Action required}
       ════════════════════════════════════════
       ```

    5. **Health verdict**:
       - Healthy: no critical/high vulnerabilities, ≤5 outdated
       - Attention needed: any high vulnerabilities OR >10 outdated
       - Action required: any critical vulnerabilities

    6. **Suggest actions**:
       - If critical CVEs: "Run {fix_command} to address critical vulnerabilities"
       - If outdated: "Run {update_command} to update {N} packages"
       - If healthy: "Dependencies look healthy!"
       - If no deps detected: "No dependency files found. DEAN supports: npm, pip, go, cargo, bundler, mix, composer"

    Error handling:
    - No project directory → "Run from a project directory"
    - Audit tool not installed → suggest install command
    - Network error → "Audit requires network access. Check connectivity."
    - Multiple ecosystems → show separate dashboard section per ecosystem
  </action>
