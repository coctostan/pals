<acceptance_criteria>

## AC-1: Hook Dispatch Completeness
```gherkin
Given kernel workflows (plan-phase.md, apply-phase.md, unify-phase.md) and module manifests (todd/module.yaml, walt/module.yaml)
When every hook declared in each module.yaml is cross-referenced against kernel dispatch steps
Then every declared hook has a corresponding dispatch point in the correct kernel workflow, and no kernel dispatch point references a hook not declared in any module
```

## AC-2: Reference File Integrity
```gherkin
Given all @-references in module workflows (plan-phase-tdd.md, apply-phase-quality.md) and module.yaml files sections
When each referenced file path is checked for existence in the repo
Then every referenced file exists at the expected relative path with no broken references
```

## AC-3: Priority Ordering Correctness
```gherkin
Given TODD pre-apply priority is 50 and WALT pre-apply priority is 100
When both modules register for the same hook (pre-apply, post-apply)
Then the kernel dispatch instructions specify ascending priority sort (lower first), ensuring TODD fires before WALT at pre-apply
```

## AC-4: Module Enable/Disable
```gherkin
Given pals.yaml with a module set to active: false
When the installer runs (dry-run analysis of install.sh logic)
Then the disabled module is skipped during installation, its hooks are not registered in modules.yaml, and its files are not copied
```

    Perform a systematic audit across 4 dimensions:

    1. **Hook Dispatch Completeness:**
       - For each hook in todd/module.yaml (pre-plan, post-plan, pre-apply, post-task, post-apply):
         verify a dispatch step exists in the correct kernel workflow
       - For each hook in walt/module.yaml (pre-apply, post-apply, post-unify):
         verify a dispatch step exists in the correct kernel workflow
       - Map: hook name → kernel workflow → step name → priority
       - Flag any missing dispatches or orphaned dispatch points

    2. **Reference File Integrity:**
       - From todd/module.yaml files.references: check each file exists under modules/todd/
       - From walt/module.yaml files.references: check each file exists under modules/walt/
       - From todd/workflows/plan-phase-tdd.md @-references: check each resolves
       - From walt/workflows/apply-phase-quality.md @-references: check each resolves
       - Flag any broken paths

    3. **Priority Ordering:**
       - Extract priorities from both module.yaml files for shared hooks (pre-apply, post-apply)
       - Verify kernel workflows specify "sort by priority ascending"
       - Confirm TODD pre-apply (50) < WALT pre-apply (100) ordering
       - Confirm post-apply ordering: WALT (100) < TODD (200)

    4. **Module Enable/Disable:**
       - Trace the pals.yaml → install.sh → drivers/claude-code/install.sh flow
       - Verify that `active: false` in pals.yaml causes the module to be skipped
       - Verify skipped modules don't appear in generated modules.yaml
       - Verify skipped module files aren't copied and commands aren't installed

    Record all findings (passes and failures) for Task 2.
