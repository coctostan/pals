<objective>
## Goal
Add GitHub Flow awareness to the three lifecycle workflows (init, resume, pause) so that project onboarding offers GitHub Flow as a first-class option, resume surfaces branch/PR/CI state with git-aware next-action routing, and pause captures git/PR continuity data in handoffs.

## Purpose
Phase 85 shipped the core loop enforcement (preflight, postflight, merge gate, transition mechanics). Those enforcement steps read `workflow: "github-flow"` config, but nothing currently *sets* that config during init, *surfaces* git state during resume, or *captures* it during pause. This phase closes the gap so users can adopt GitHub Flow during onboarding and see git state throughout the lifecycle.

## Output
- `kernel/workflows/init-project.md` — updated `configure_git` step with GitHub Flow option + `gh` validation
- `kernel/workflows/resume-project.md` — new `check_git_state` step + git-aware routing in `determine_single_action`
- `kernel/workflows/pause-work.md` — updated handoff template with git/PR continuity + `workflow` config resolution
</objective>

<task type="auto">
  <name>Task 2: Add git state surfacing and git-aware routing to resume-project</name>
  <files>kernel/workflows/resume-project.md</files>
  <action>
    Update `kernel/workflows/resume-project.md`:

    1. **Add a new step `check_git_state`** between `load_resume_context` and `determine_single_action`:

       ```xml
       <step name="check_git_state">
       **Surface git/PR state for github-flow projects.**

       1. Read pals.json and resolve GIT_WORKFLOW using 3-tier resolution:
          - If `git.workflow` exists → use it
          - Else if `git.branching` exists → "legacy"
          - Else → "none"

       2. If GIT_WORKFLOW = "github-flow":
          a. Detect current branch:
             ```bash
             git branch --show-current
             ```
          b. Read GIT_BASE_BRANCH from pals.json `git.base_branch` (default: "main")
          c. Check ahead/behind base:
             ```bash
             git rev-list --left-right --count origin/{GIT_BASE_BRANCH}...HEAD
             ```
          d. Check for open PR:
             ```bash
             gh pr view --json url,state,statusCheckRollup 2>/dev/null
             ```
          e. If PR exists, extract:
             - PR URL
             - PR state (OPEN, MERGED, CLOSED)
             - CI state from statusCheckRollup (all SUCCESS → "passing", any FAILURE → "failing", else "pending")
          f. Store: CURRENT_BRANCH, GIT_BASE_BRANCH, PR_URL, PR_STATE, CI_STATE, AHEAD_COUNT, BEHIND_COUNT

       3. If GIT_WORKFLOW != "github-flow": skip entirely (no git state surfacing)
       </step>
       ```

    2. **Update `determine_single_action`** to add git-aware routing BEFORE the existing loop-position routing:

       Add this block at the top of `determine_single_action`:
       ```
       **Git-aware routing (github-flow only):**
       If GIT_WORKFLOW = "github-flow" AND git state was collected:

       | Git State | Override Next Action |
       |-----------|---------------------|
       | PR open + CI failing | "Fix CI failures, then merge PR" |
       | PR open + CI passing + reviews pending (if require_reviews) | "Get PR reviewed" |
       | PR open + CI passing + ready to merge | "Merge PR: `gh pr merge`" |
       | No PR + loop complete (all ✓) | Normal routing (next PLAN will create branch) |

       If no git override applies, fall through to the existing loop-position routing.
       ```

    3. **Update `report_and_route`** display template to include git state:

       After the Loop Position block and before "Last Session:", add:
       ```
       {If GIT_WORKFLOW = "github-flow":}
       Git State:
       ┌─────────────────────────────────────┐
       │  Branch: {CURRENT_BRANCH}           │
       │  Base:   {GIT_BASE_BRANCH}          │
       │  PR:     {PR_URL} ({PR_STATE})      │
       │  CI:     {CI_STATE}                 │
       │  Sync:   {BEHIND_COUNT behind / AHEAD_COUNT ahead} │
       └─────────────────────────────────────┘
       ```
       If no PR exists, show "PR: none" instead.
       If GIT_WORKFLOW is not github-flow, omit the entire Git State block.

    **DO NOT** change `verify_paul_exists`, `detect_handoffs`, or `handoff_lifecycle` steps.
  </action>
