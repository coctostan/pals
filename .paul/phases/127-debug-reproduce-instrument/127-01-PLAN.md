    - Apply the same pattern to `kernel/workflows/fix-loop.md` dispatch steps.

    **Process Failure #4 — PLAN template has no module section:**
    - In `kernel/templates/PLAN.md`: Add a "Pre-Plan Module Dispatch" section between `</context>` and `<skills>` (or after skills if skills is absent):

      ```markdown
      </module_dispatch>
      ```

    **Process Failure #5 — SUMMARY template module section is optional:**
    - In `kernel/templates/SUMMARY.md` line ~77: Replace "If no module evidence survives finalization, omit this section entirely." with "REQUIRED. If no modules fired, state why: registry not loaded, no hooks registered for this phase, or modules read but produced no output."
