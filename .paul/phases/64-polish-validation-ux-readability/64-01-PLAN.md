    3. **Loop badge in status bar** — add a `renderLoopBadge` function after `parsePalsState`:
       ```typescript
       function renderLoopBadge(loopString?: string): string | undefined {
         if (!loopString) return undefined;
         const planDone = /✓\s*○\s*○|✓\s*✓|✓[^○]*✓/.test(loopString) || /PLAN.*✓/.test(loopString);
         const applyDone = /✓\s*✓\s*○|✓\s*✓\s*✓/.test(loopString);
         const unifyDone = /✓\s*✓\s*✓/.test(loopString);
         const planMark = planDone ? "✓" : "○";
         const applyMark = applyDone ? "✓" : "○";
         const unifyMark = unifyDone ? "✓" : "○";
         return `PLAN${planMark} APPLY${applyMark} UNIFY${unifyMark}`;
       }
       ```
       The regex logic: presence of first ✓ = plan done; second ✓ = apply done; third ✓ = unify done.
       A simpler approach: count ✓ characters in loopString.
       ```typescript
       function renderLoopBadge(loopString?: string): string | undefined {
         if (!loopString) return undefined;
         const marks = [...loopString.matchAll(/[✓○]/g)].map(m => m[0]);
         if (marks.length < 3) return undefined;
         return `PLAN${marks[0]} APPLY${marks[1]} UNIFY${marks[2]}`;
       }
       ```
       Use the count-based approach (simpler and less fragile).

    4. **Inject loop badge into status bar** — in `renderLifecycleStatus`, add the badge after the phase:
       ```typescript
       function renderLifecycleStatus(state: PalsStateSnapshot): string | undefined {
         if (!state.detected) return undefined;
         return [
           "PALS",
           state.phase ? `Phase: ${state.phase}` : null,
           renderLoopBadge(state.loop),           // add this line
           state.nextAction ? `Next: ${state.nextAction}` : null,
           renderQuickActionSummary(state),
         ]
           .filter(Boolean)
           .join(" • ");
       }
       ```

    What to avoid:
    - Do NOT change any other notify levels (warning for "no next action" is already correct)
    - Do NOT add colors to the widget lines (not supported by Pi setWidget)
    - Do NOT change routing logic, activation model, or guided workflow detection
    - Keep the loop badge purely additive; if marks.length < 3, return undefined gracefully
  </action>
