<objective>
## Goal
Extract the S7 `pals-context-injection` subsystem from `drivers/pi/extensions/pals-hooks.ts` into a new sibling Pi extension module `drivers/pi/extensions/pals-context-injection.ts` without changing PALS context payload, injection gating, message-stream housekeeping, or `/paul-resume` consumer-visible authority/activation tag behavior.

## Purpose
This continues the v2.59 sibling Pi extension extraction wave after Phase 253 established the S7 contract. The extraction reduces `pals-hooks.ts` responsibility while preserving the exact byte sequences `[PALS_STATE_AUTHORITY=.paul/STATE.md]` and `[PALS_ACTIVATION_SIGNAL]`, the legacy-header migration path `## PALS Context (auto-injected)`, the custom-message-type identity `pals-context`, the `before_agent_start` / `context` Pi events the injection currently fires on, and the message-housekeeping invocation site (filter `isLegacyPalsContextMessage` → normalize via `keepOnlyLatestPalsContextMessage` → gate via `messagesChanged`). It also preserves `.paul/*` authority, transcript-visible canonical replies, no inferred merge intent, no auto-approval, no auto-continue, no skipped checkpoints, no UI-only lifecycle decisions, and command-output validation truth.

## Output
- New runtime sibling: `drivers/pi/extensions/pals-context-injection.ts` exporting the six S7 functions and defining the six S7 constants single-defined.
- Updated `drivers/pi/extensions/pals-hooks.ts`: imports the moved S7 functions/constants from `./pals-context-injection`; keeps S4 canonical-reply delivery inline; keeps shared helpers (`extractTextContent`, `collectRecentAssistantTexts`, `compactWhitespace`, `readFileOr`, `getFileFreshness`, `selectBoundedLines`, `escapeRegExp`, `PalsStateSnapshot`, `ActivationState`) defined and exported; keeps the message-housekeeping invocation site functionally identical.
- Updated `tests/pi-end-to-end-validation.sh`: one localized Pi-supported-runtime guardrail block proving the new sibling holds the six S7 constants + six S7 functions + loader-compat marker + housekeeping behavior, and that `pals-hooks.ts` no longer declares them inline while still keeping S4 inline and shared helpers intact.
</objective>
