## Goal
Surface the Phase 223 Pi-first install/uninstall cleanup and the current post-v2.51 validation baseline in user-facing docs, Pi-facing docs, historical validation-report artifacts, and localized validation guardrails.

Phase 224 makes that posture discoverable and command-checked without changing runtime behavior, installer behavior, validation semantics, dependency manifests, CI, GitHub Flow automation, helper delegation semantics, or lifecycle authority.

## Output
- README and Pi-facing docs state the current Pi-first install/uninstall posture, explicit `PALS_DRIVER=...` opt-ins, and current command-output validation baseline.
- Historical validation reports stay archival/non-authoritative but route readers to fresh commands and the current Pi 203/203 + cross-harness 119/119 baseline.
- The legacy-retention cleanup contract records Phase 223 implementation evidence and Phase 224 surfacing expectations.
- The validation-classification contract no longer leaves Phase 221 as merely a target; it records v2.51 closure as complete and hands off to v2.52 cleanup surfacing.
- Pi validation is expected to increase from 203→204 and cross-harness validation from 119→120 only because each suite gains one localized docs/install-surfacing guardrail.
