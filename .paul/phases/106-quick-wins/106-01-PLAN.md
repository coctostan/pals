## Goal
Implement 4 quick-win module improvements: SKIP structured templates (AI-1), DEAN severity blocking (AI-6), DOCS expanded catalog (AI-5), TODD TDD suggestion (AI-8).

## Purpose
These are small-effort, high-impact changes to hook descriptions and reference files that transform module output from generic to actionable.

## Output
- SKIP produces structured Decision Record entries instead of "decision found"
- DEAN blocks on critical/high severity, warns on moderate
- DOCS checks CHANGELOG, API docs, and proximity-based drift beyond README
- TODD suggests TDD plan type when test coverage exists for planned changes

<task type="auto">

  <action>
    Update DOCS pre-plan description from:
    "Scan for project docs (README, CHANGELOG, API docs, config docs); inject doc_warnings with stale/missing doc counts into plan context"
    to:
    "Scan for project docs using expanded catalog: README.md, CHANGELOG.md, CONTRIBUTING.md, docs/ directory, API.md, any *.md in project root.     "Compare files_changed against doc catalog; warn if code changed but nearby docs not updated; surface drift warnings"
    to:
    "Compare files_changed against expanded doc catalog. For each changed source file, check: 1) Was README.md updated? 2) Was CHANGELOG.md updated? 3) If docs/ directory exists, was the corresponding doc updated? 4) Proximity check: if src/X.ts changed, did docs/X.md or nearby docs change? Surface specific drift warnings with file pairs: 'src/verify.ts changed but docs/verify.md was not updated'."
  </action>
  <verify>grep "CHANGELOG\|CONTRIBUTING\|proximity\|docs/ directory\|file pairs" modules/docs/module.yaml</verify>
