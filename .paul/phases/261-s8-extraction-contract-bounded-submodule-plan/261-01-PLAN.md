<objective>
## Goal
Create the v2.61 Phase 261 S8 `command-routing` extraction contract and bounded Phase 262 handoff before any runtime/source extraction work begins.

## Purpose
PALS is continuing the Pi extension sibling-module extraction wave after S5 (Phase 239), S1 (Phase 243), S2 (Phase 246), S3 (Phase 250), S7 (Phase 254), and S6 (Phase 258) shipped. S8 owns the user-facing Pi `/paul-*` command surface: command-definition table, command lookup, wrapper-command translation, command-signal detection, quick-action selection and bounding, command-activation marking, and command/shortcut registration on Pi. This subsystem is the **highest user-visible blast radius** of any remaining extraction candidate (it defines the entire `/paul-*` entry surface, the Ctrl+Alt quick-action shortcuts, and how `pi.sendUserMessage` is invoked to deliver canonical `/skill:*` routing) and is the second runtime-coupled sibling extraction (registers commands and shortcuts on Pi rather than purely producing strings). This plan turns the v2.60 closure recommendation candidate (recorded in `.paul/phases/260-evidence-closure-next-roadmap-decision-v2-60/260-01-SUMMARY.md` and explicitly approved through `/paul:milestone` Q1=a) into an executable contract boundary while preserving the eleven `/paul-*` command names, the registration call shapes (`pi.registerCommand(name, { description, handler })`), the quick-action shortcut registration (`pi.registerShortcut`) and bounded layer (`PRIMARY_QUICK_ACTION_LIMIT = 3`, `MAX_QUICK_ACTIONS = 5`, five distinct `Key.ctrlAlt("[nsrhm]")` shortcuts), the canonical-skill routing behavior (`/paul-*` → `/skill:paul-*` via `pi.sendUserMessage`), the `markActivation` activation-state coupling, the no-UI-only-lifecycle-decisions boundary, S4 canonical reply delivery, `.paul/*` artifact authority, transcript-visible canonical replies, no inferred merge intent, GitHub Flow gates, parent-owned APPLY, module evidence, and command-output validation truth.

## Output
- New docs-only contract: `docs/PI-NATIVE-COMMAND-ROUTING-EXTRACTION-CONTRACT.md`.
- Updated extraction target baseline: `docs/PI-NATIVE-EXTENSION-EXTRACTION-TARGET-BASELINE.md` records the Phase 261 S8 promotion and Phase 262 handoff while preserving S1/S2/S3/S5/S6/S7 history verbatim.
