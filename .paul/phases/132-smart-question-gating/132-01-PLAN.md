<objective>
## Goal
Add conditional question gating to init-project.md so that irrelevant questions are skipped based on prior answers (no GitHub → skip PR/CI questions; greenfield with no source files → skip brownfield mapping; skip SonarQube and specialized skills questions for all projects).

## Purpose
v2.25 adversarial testing (Items 6, 7, 11) showed that a greenfield project with no GitHub repo was asked about SonarQube, specialized skills, worktrees, auto-PR, and CI checks — all irrelevant. Contradictory questions (auto-PR after choosing "no GitHub") undermine user trust and create onboarding fatigue.

## Output
Updated `kernel/workflows/init-project.md` with conditional logic in three areas: brownfield detection, git automation questions, and integration/skills questions.
</objective>
