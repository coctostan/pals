    2. In `modules.yaml`, update DOCS's `post-apply` hook description from `">-"` to explicit instruction:
       "Compare files_changed against doc_catalog from pre-plan. For each changed code directory, check proximity-mapped docs using rules from doc-drift.md. Flag as DRIFT if related doc exists but was not updated. Flag as COVERAGE GAP if significant code changes have no related docs. Report per-file: path, status (DRIFT/OK/GAP), related code files."

    3. In `modules/docs/references/doc-drift.md`, add a "Hook Output Requirements" section:
       - DOCS post-apply MUST check all doc files in catalog, not just README.md

       - DRIFT = doc exists but not updated when related code changed
       - OK = doc was updated or no related code changed
       - GAP = code changed significantly but no related doc exists
       - Example of GOOD output (multi-file table) vs BAD output (README-only check)

    4. In `modules/docs/references/doc-detection.md`, ensure the proximity mapping section is prominent and includes explicit instructions for the hook to USE the mapping (not just document it).
