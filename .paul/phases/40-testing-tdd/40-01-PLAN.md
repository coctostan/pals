<objective>
## Goal
Deep-dive how 10+ agentic coding frameworks approach testing and TDD — test generation, coverage targeting, RED/GREEN/REFACTOR enforcement, test-first vs test-after, failure diagnosis, and test quality. Produce a findings document with adopt/adapt/reject decisions, then implement concrete improvements to TODD's module files.

## Purpose
TODD enforces TDD with a solid RED→GREEN→REFACTOR cycle, but was designed from first principles. Aider runs tests after every edit. Superpowers mandates 5-step verification per chunk. SWE-agent has test-driven repair. We haven't systematically compared TODD against what works elsewhere. This phase fixes that — strengthening TODD with the best techniques from across the ecosystem.

## Output
1. `testing-research.md` — deep competitive findings with specific techniques cited
2. Modified TODD module files — concrete improvements adopted from research
</objective>
