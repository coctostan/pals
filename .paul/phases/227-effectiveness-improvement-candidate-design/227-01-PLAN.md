# Phase 227 Plan 01: Artifact-Slice Targeting Candidate Design

<objective>

## Goal

Create a source-cited candidate design for the highest-value v2.53 Pi integration improvement: artifact-slice targeting and deduplication. The design must define the smallest safe Phase 228 spike, expected repo surfaces, evidence requirements, validation classes, fallback rules, and explicit non-goals without implementing runtime behavior.

## Purpose

Phase 226 identified artifact-slice targeting and deduplication as the leading efficiency/effectiveness hotspot because it can reduce repeated STATE/ROADMAP/PLAN orientation context while preserving `.paul/*` artifact authority, command-output truth, lifecycle gates, module evidence, parent-owned APPLY, GitHub Flow safety, and explicit approval/checkpoint behavior. Phase 227 turns that discovery into an executable design handoff for Phase 228.

- A selected candidate and a bounded Phase 228 spike scope.
- No runtime/source/test/install/dependency/CI/GitHub Flow/helper/lifecycle-authority changes.
- No validation-script changes in this plan.
