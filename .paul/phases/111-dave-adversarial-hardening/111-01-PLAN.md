  <name>Task 1: Rewrite DAVE hook descriptions for A-grade output</name>
  <files>modules/dave/module.yaml</files>
  <action>
    Rewrite both DAVE hook descriptions to be maximally explicit:
    
    For pre-plan hook:
    1. Run: find . -name '*.yml' -path '*/.github/workflows/*' -o -name '.gitlab-ci.yml' -o -name 'Dockerfile' -o -name 'docker-compose.yml' -o -name 'Jenkinsfile' -o -name '.circleci/config.yml'
    2. Output a CI inventory table: '| File | Platform | Status |' listing each found config
    3. If NO CI config found: suggest generating .github/workflows/ci.yml from pipeline-generation.md templates with the specific project type detected (Node.js/Python/Go/Rust based on lock files)
    4. If CI found: check if it covers: build, test, lint steps — warn if any missing
    5. Inject deploy_context with: ci_files_found, ci_platforms, coverage_gaps
    
    For post-apply hook:
    1. For EACH modified CI config file, run YAML validation: cat {file} | python3 -c "import sys,yaml; yaml.safe_load(sys.stdin)" or yq '.' {file} — capture stderr for parse errors with line numbers
    2. For GitHub Actions files, grep for required keys: 'runs-on', 'uses: actions/checkout', 'run:' containing test/lint commands
    3. For Dockerfile: grep 'FROM' and check if tag is pinned (not :latest, not bare image name)
    4. Output a validation table: '| File | Check | Result |' with PASS/FAIL/WARN per check
    5. BLOCK on any YAML parse error — display: 'DAVE BLOCK: {file} has YAML syntax error at line {N}: {message}'
    6. WARN on missing best practices (no test step, unpinned FROM, no cache)
    7. If no CI files were modified: skip gracefully with 'No CI config files in changed set'
  </action>
  <verify>
    grep -c "BLOCK\|validation table\|YAML.*error\|parse error\|runs-on\|checkout\|Dockerfile\|FROM.*pinned\|inventory\|coverage_gaps" modules/dave/module.yaml

  </verify>
