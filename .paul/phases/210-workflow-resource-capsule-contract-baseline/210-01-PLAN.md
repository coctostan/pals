<objective>
## Goal
Create the v2.49 workflow/resource capsule contract baseline: define which installed resources are eligible for orientation capsules, what schema and citation fields every capsule must expose, which blocking workflow semantics must be excluded from capsules, when full authoritative reads are mandatory, and what Phase 211+ runtime/docs/validation handoffs must prove.

## Purpose
v2.47 hardened artifact slices and v2.48 hardened guided workflow evidence. v2.49 can reduce repeated workflow/resource orientation cost only if capsules are explicitly limited to derived, source-cited orientation aids over installed resources and cannot replace full workflow reads for STOP/DO NOT SKIP rules, checkpoints, lifecycle writes, GitHub Flow gates, module enforcement, approvals, or command-output validation truth.

## Output
- New `docs/PI-NATIVE-WORKFLOW-RESOURCE-CAPSULE-CONTRACT.md` contract baseline.
- A named v2.49 handoff for Phase 211 runtime capsule loading hardening, Phase 212 docs/validation surfacing, and Phase 213 evidence reconciliation/milestone closure.
- Completion evidence in `210-01-SUMMARY.md` after APPLY/UNIFY.
</objective>
