<objective>
## Goal
Add lightweight artifact consistency guardrails that detect drift among `.paul/STATE.md`, `.paul/ROADMAP.md`, `.paul/MILESTONES.md`, latest phase SUMMARY metadata, and session-continuity handoff/resume references without creating a new source of lifecycle truth.

## Purpose
Phase 234 identified lifecycle/index artifact drift as weakness #3: PALS depends on durable `.paul/*` truth, but current milestone, phase, loop, summary, and handoff state is repeated across several hot artifacts. Phase 235 requires artifact authority to remain in `.paul/*` and forbids hidden state, telemetry, or runtime-owned ledgers. Phase 236 preserved validation command truth while beginning validation-suite decomposition. This plan adds a small, explicit guardrail layer so future transitions can catch stale index/session pointers quickly while preserving `.paul/*` authority and command-output validation truth.

## Output
- New `docs/PALS-ARTIFACT-CONSISTENCY-GUARDRAILS.md` documenting the guardrail invariants, tier-aware scope, protected authority boundaries, and non-goals.
- New `tests/helpers/artifact_consistency.sh` containing a source-safe helper/checker for current-state, milestone/index, summary-frontmatter, resume-file, and active-handoff consistency.
- Updated `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` with one localized artifact-consistency guardrail each.
- Updated `.paul/MILESTONES.md` current/future pointer to the current v2.55 Phase 237 state so the new guardrail starts from a clean baseline.
- Expected validation count changes reconciled as guardrail additions only: Pi validation `209/209` → `210/210`; cross-harness validation `123/123` → `124/124`.
</objective>
