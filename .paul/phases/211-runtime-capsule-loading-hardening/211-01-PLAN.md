<objective>
## Goal
Harden the repo-source Pi runtime so explicit PALS context activation can include a small, source-cited workflow/resource capsule block that orients models to installed workflow/reference resources while preserving full authoritative reads for all consequential workflow actions.

## Purpose
Phase 210 defined workflow/resource capsules as installed-resource orientation aids only. Phase 211 proves the runtime half of that contract in `drivers/pi/extensions/pals-hooks.ts` without changing validation scripts, docs surfacing, dependency manifests, CI, helper delegation, legacy surfaces, or lifecycle authority.

## Output
- `drivers/pi/extensions/pals-hooks.ts` gains a bounded workflow/resource capsule inventory and renderer.
- The bounded PALS context payload emits capsule schema markers only after the existing explicit `/paul-*` or `/skill:paul-*` activation signal.
- Runtime output includes `Capsule:`, `Source:`, `Source type:`, `Freshness:`, `Bounds:`, `Fallback:`, and `Derived aid only` markers.
- Runtime code remains read-only/disposable: no `.paul/*` writes, hidden persistence, telemetry, cache-as-truth, module ledger, validation ledger, or lifecycle ledger.
</objective>
