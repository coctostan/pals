# Phase 267 Plan 01: Selective Artifact Loading + Anti-Regrowth Guardrails + v2.62 Closure

<objective>
## Goal
Implement the Phase 267 v2.62 closure slice: add selective artifact loading contracts to the highest-impact PALS workflows, protect against regrowth with localized validation guardrails, and close v2.62 with lifecycle evidence plus a recommended-not-approved v2.63 candidate.

## Purpose
Phase 266 made the four hot `.paul/*` artifacts small enough for the v2.62 budget, but it did not change PALS behavior. This plan makes the workflow contract explicit: hot lifecycle artifacts should be located by relevant heading, marker, row, phase, plan ID, resume file, or status line first, then read through bounded windows by default. Full-file reads remain available, but only as a deliberate fallback for rewrite, audit, repair, migration, or whole-artifact validation.

## Output
- Eight repo source workflows carry the selective-loading rule without tool-compatibility bloat.
- Validation suites gain localized anti-regrowth TAPs for hot artifact budgets, archive pointers, selective-loading wording, and the 7th closure-guardrail recipe application.
- `.paul/MILESTONES.md`, `.paul/PROJECT.md`, and `.paul/ROADMAP.md` record v2.62 closure evidence and a v2.63 recommendation.
- UNIFY creates `.paul/phases/267-validation-guardrails-evidence-closure/267-01-SUMMARY.md`.
</objective>
