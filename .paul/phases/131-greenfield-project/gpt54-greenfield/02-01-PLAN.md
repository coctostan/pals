<objective>
## Goal
Extend `pi-bench` so benchmark runs are persistently saved, result presentation is improved for ongoing comparison use, and users can inspect prior runs through a `/bench history` subcommand.

## Purpose
This advances pi-bench from a one-shot benchmark command toward a reusable comparison workflow by keeping results, presenting them more cleanly, and letting users review previous runs without rerunning benchmarks.

## Output
- Persistent benchmark history storage in `.bench-history.json`
- Updated `/bench` flow that formats results as richer markdown table output and saves each run
- `/bench history` subcommand path for viewing prior saved runs
- Automated tests and README documentation for persistence and history usage
</objective>

