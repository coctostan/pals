<objective>
## Goal
Create the DAVE (Deploy Automation & Verification Engine) module as a new first-party pal with a `/paul:deploy` command that generates CI/CD pipelines, validates configurations, and guides post-deploy verification.

## Purpose
PALS manages the development lifecycle (plan/apply/unify) but has no visibility into deployment pipelines. Projects accumulate misconfigured or missing CI/CD silently. DAVE makes deployment health visible — generating GitHub Actions, validating pipeline configs, and providing post-deploy smoke test guidance. This is the 7th pal, continuing to prove module system scalability.

## Output
- `modules/dave/module.yaml` — module manifest (no hooks initially, Phase 25 adds hooks)
- `modules/dave/references/deploy-principles.md` — deployment philosophy, CI/CD best practices, pipeline patterns
- `modules/dave/references/pipeline-generation.md` — templates and generation patterns for GitHub Actions, GitLab CI, etc.
- `modules/dave/references/deploy-verification.md` — post-deploy smoke tests, health checks, rollback criteria
- `kernel/commands/paul/deploy.md` — `/paul:deploy` command (audit + generate + verify modes)
</objective>
