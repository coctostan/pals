<objective>
## Goal
Broaden delegated APPLY eligibility so more bounded autonomous tasks can route through `pals-implementer` under explicit parent control, without weakening verification, module enforcement, fallback, or `.paul/*` lifecycle authority.

## Purpose
Phase 156 introduced the first parent-controlled delegated APPLY path, and Phase 164 aligned the config contract that enables it. This phase should make the path practical for more real APPLY tasks by refining the kernel-side eligibility model and proving the widened behavior across repo and installed validation surfaces.

## Output
A Phase 165 implementation that:
- expands delegated APPLY eligibility semantics for bounded autonomous tasks in the canonical APPLY workflow
- aligns the shared subagent criteria and repo-owned `pals-implementer` / Pi apply skill wording with the broader parent-controlled operating model
- adds validation coverage that catches drift in the widened delegation contract without blurring REV or lifecycle authority
</objective>
