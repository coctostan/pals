<objective>
## Goal
Create a thin Pi TypeScript extension that maps Pi lifecycle events to PALS module hook dispatch and registers `/paul-*` commands via `pi.registerCommand()`.

## Purpose
Pi has no equivalent to Claude Code's shell hooks. This extension bridges the gap by listening to Pi events (`before_agent_start`, `tool_call`, `tool_result`, `agent_end`) and mapping them to PALS hook points (pre-plan, pre-task, post-task, post-apply). It also registers slash commands so Pi users can invoke PALS workflows without typing `/skill:paul-*`.

## Output
- `drivers/pi/extensions/pals-hooks.ts` — the Pi extension
- `drivers/pi/extensions/README.md` — documentation for the extension
</objective>

## Pi Extension API (from research)
Extensions are TypeScript modules exporting `default function(pi: ExtensionAPI)`:
- Events: `session_start`, `before_agent_start`, `agent_start`, `agent_end`, `turn_start`, `turn_end`, `tool_call` (can block), `tool_result` (can modify), `context` (modify messages)
- `pi.registerTool()` — custom tools with TypeBox schemas
- `pi.registerCommand()` — slash commands
- `pi.appendEntry()` — persistent state across restarts

## PALS Hook Points (from docs/HOOK-CONTRACT.md)
- pre-plan: before plan creation
- post-plan: after plan creation
- pre-apply: before task execution
- post-task: after each task
- post-apply: after all tasks
- pre-unify: before reconciliation
- post-unify: after reconciliation
