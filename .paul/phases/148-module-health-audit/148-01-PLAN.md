<task type="auto">
  <name>Task 2: Grade Effectiveness and Produce Recommendations</name>
  <files>.paul/phases/148-module-health-audit/module-health-scorecard.md</files>
  <action>
    For all 18 modules, grade effectiveness on three dimensions:
    
    1. **Relevance** (A-D): How often does this module's concern apply to typical PALS projects?
       - A = almost every project (tests, quality, deps)
       - B = most projects (security, docs, architecture)
       - C = some projects (UI, API, data, deploy)
       - D = niche (rarely applicable)
    
    2. **Actionability** (A-D): When fired, does it produce output the developer can act on?
       - A = specific file:line findings with fix guidance
       - B = useful warnings with clear next steps  
       - C = generic advisories, often skipped
       - D = noise — rarely produces useful output
    
    3. **Distinctness** (A-D): How unique is this module's contribution?
       - A = no overlap, unique domain
       - B = minor overlap, distinct enough to justify
       - C = significant overlap, could merge
       - D = fully redundant with another module
    
    Use evidence from:
    - v2.25 E2E data (which modules fired, quality of output across 3 models)
    - v2.29 E2E report (Sonnet dispatch evidence)
    - Hook descriptions (what they actually check)
    - Reference file content (depth of guidance)
    
    Then produce recommendations per module:
    - **KEEP** — earning its place, no changes needed
    - **TRIM** — good module but bloated reference files or descriptions; reduce context cost
    - **MERGE** — combine with another module (specify which)
    - **CUT** — not earning its context cost; disable or remove
    
    Finally, write a prioritized action list for Phase 149.
    
    Append to module-health-scorecard.md: Effectiveness Grades table, Recommendations section, Phase 149 Action List.
  </action>
  <verify>
    - All 18 modules have grades across 3 dimensions
    - Every module has a KEEP/TRIM/MERGE/CUT recommendation with rationale
    - Phase 149 action list is ordered by impact
    - Scorecard has summary statistics (total context cost, modules per recommendation category)
  </verify>
  <done>AC-3, AC-5 satisfied</done>
