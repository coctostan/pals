<objective>
## Goal
Create the SKIP (Smart Knowledge & Information Persistence) module as a new first-party pal with a `/paul:knowledge` command that captures, indexes, and surfaces project decisions, rationale, and context.

## Purpose
PALS tracks *what* happened (STATE.md, SUMMARYs, git history) but not *why* decisions were made or the context behind them. SKIP makes institutional knowledge searchable — the same way TODD made testing visible, WALT made quality visible, DEAN made dependency health visible, and IRIS made review patterns visible. This proves the module system scales to 6 pals.

## Output
- `modules/skip/module.yaml` — module manifest (no hooks in v0.7 initially, hooks come in Phase 25)
- `modules/skip/references/knowledge-principles.md` — what to capture, when, and how
- `modules/skip/references/knowledge-capture.md` — capture patterns and templates for decisions, rationale, trade-offs
- `modules/skip/references/knowledge-search.md` — search/retrieval heuristics for surfacing relevant knowledge
- `kernel/commands/paul/knowledge.md` — `/paul:knowledge` command (capture + search modes)
</objective>

    - Guidelines: capture the "why" not the "what" (code captures the what)

    knowledge-search.md:
    - Search strategies: keyword in KNOWLEDGE.md, grep project decisions from STATE.md accumulated context, scan SUMMARY files for decisions
    - Filters: by date range, by phase/milestone, by knowledge type, by keyword
    - Retrieval: present matches with surrounding context, link to source files
    - Heuristics: when a user asks "why did we...", search decisions; when "what alternatives...", search trade-offs
