<objective>
## Goal
Implement the calmer Pi lifecycle presentation defined in Phase 167 by simplifying the persistent widget/status render path, removing persistent quick-action clutter, and adding bounded semantic emphasis where the Pi surfaces support it cleanly.

## Purpose
Phase 167 converted the user complaint into an explicit runtime target: the current Pi lifecycle surface already has the right core fields, but it is too noisy because shortcut hints, module detail, and redundant chrome compete with milestone, phase, loop, and next action. Phase 168 is the implementation phase that turns that design package into the actual runtime rendering behavior in `drivers/pi/extensions/pals-hooks.ts`.

## Output
An updated `drivers/pi/extensions/pals-hooks.ts` whose persistent lifecycle status/widget surfaces reflect the calmer 4-line model and reduced secondary clutter without changing lifecycle authority, command routing semantics, or the guided workflow contract.
</objective>
