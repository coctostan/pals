<objective>
## Goal
Implement the first bounded safe cleanup from the Phase 222 contract by making the root install/uninstall surface Pi-first by default and explicit about frozen legacy opt-in, without touching Pi driver runtime files, validation scripts, dependency manifests, CI, GitHub Flow automation, helper delegation semantics, or lifecycle authority.

## Purpose
Phase 222 established that legacy/install-surface cleanup may proceed only where Shared Invariant, Artifact / Process, and Pi-Supported Runtime guarantees remain protected. The root `install.sh` and `uninstall.sh` are public install-surface entry points, and README already states Pi is the supported runtime while Claude Code and Agent SDK are unsupported/frozen historical surfaces. This plan aligns the root installer/uninstaller behavior and docs with that support posture: Pi stays the default supported path, legacy drivers remain repo source and explicit opt-in surfaces, and current command-output validation remains authoritative.

## Output
Update the root installer/uninstaller and README install section so default install/uninstall routing prefers Pi when available, legacy driver actions are explicit opt-in through `PALS_DRIVER=claude-code` / `PALS_DRIVER=agent-sdk` or an explicitly documented all-drivers mode, and validation proves Pi install behavior plus current Pi/cross-harness suites remain green.
</objective>
