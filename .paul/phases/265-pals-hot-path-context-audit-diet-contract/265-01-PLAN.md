<objective>
## Goal
Create `docs/PALS-STATE-CONTEXT-DIET-CONTRACT.md`, the v2.62 Phase 265 docs-only audit + contract baseline that defines hot/warm/cold split, archive structure, per-section live-byte budgets, and Phase 266/267 handoff for compacting `.paul/STATE.md` and `.paul/PROJECT.md` without weakening lifecycle authority, helper compatibility, or full authoritative read fallback.

## Purpose
v2.62 applies the v2.43/v2.44 Context Diet archive-and-compact recipe to STATE.md and PROJECT.md, which were never given the same treatment. Pain that triggered the milestone: `/paul:resume` consumes ~50 KB of context per session against current `.paul/*` artifacts (STATE.md alone is 99 KB; PROJECT.md is 67 KB). Phase 265 establishes the evidence base and contract before Phase 266 mutates `.paul/*` artifacts and Phase 267 adds anti-regrowth validation guardrails. Without a contract baseline, Phase 266 risks drift in `tests/helpers/artifact_consistency.sh`, validation-script structural assertions, Pi runtime expectations, CARL session-boundary expectations, or full authoritative read fallback.

## Output
A single docs-first contract artifact with:
- baseline measurement of hot-path read total (byte counts for STATE/PROJECT/MILESTONES/ROADMAP and the actual `/paul:resume` workflow read order)
- per-file active-vs-cold split audit for STATE.md, PROJECT.md, MILESTONES.md, ROADMAP.md
- cross-file duplication record (deferred fix; no remediation in v2.62)
- preservation invariants inventory (artifact_consistency_check helper deps, Pi runtime deps, CARL deps, validation-script deps, full authoritative read fallback)
- defined hot/warm/cold split rules, archive structure, per-section live-byte budgets (STATE.md `live_summary` / `decisions` / `fixes` / `blockers` / `git_state`; PROJECT.md `active_summary` / `milestone_posture` / `key_decisions`), and aggregate hot-path resume read target (≤ 80 KB)
- Phase 266 / Phase 267 handoff (allowed scope, expected validation count change, archive file naming, recipe-application guidance)
</objective>
