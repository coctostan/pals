<objective>
## Goal
Audit and rewrite the Phase 277 backend/runtime cross-cutting domain optional module cohort — ARCH, SETH, OMAR, PETE, REED, and VERA manifest hook descriptions plus their primary `*-patterns.md` operational references and ARCH's structural drift/onboarding references — so domain advisory guidance remains useful, compact, bounded, source-grounded, non-blocking, and free of stale ceremony, marker-only evidence, broad search/read commands, magical inference, or lifecycle-authority drift.

## Purpose
Phase 277 applies the v2.63/Phase 275/Phase 276 collaborative instruction-audit pattern to the domain optional module cohort: audit first, then walk files one by one with visible before/after proposed edits before any mutation. This first 277 plan handles the backend/runtime cross-cutting half of the cohort (architecture, security, observability, performance, resilience, privacy) so domain advisory surfaces improve PLAN/APPLY/UNIFY evidence without claiming lifecycle authority, blocking progression, reading broad context by default, inventing source facts, or persisting bulky/ambiguous artifacts. The UI/UX/data/deploy/DEAN-deferred half (ARIA, LUKE, GABE, DANA, DAVE, DEAN-deferred-refs) is reserved for a follow-up Phase 277 plan.

## Output
A bounded domain-module rewrite with:
- compact audit of ARCH, SETH, OMAR, PETE, REED, and VERA approved files plus relevant deferred principle/onboarding files;
- blocking user approval checkpoint before source edits;
- approved file-by-file, section-by-section collaboration loop: within each file, handle one section/edit at a time by showing the current excerpt (`BEFORE`), concrete problem, proposed replacement/focused diff (`AFTER`), affected advisory/persistence semantics, verification expectation, user approval or wording change, then edit only that approved section;
- re-read verification after each approved section edit before moving to the next section in the same file or the next file;
- final validation evidence showing domain modules do not block, persistent advisory artifacts have explicit destinations/schemas, broad or magical inference is removed, lifecycle authority stays with PLAN/APPLY/UNIFY, and scope stayed within approved instruction files plus routine `.paul/*` lifecycle artifacts.
</objective>
