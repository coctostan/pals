<objective>
## Goal
Make install.sh upgrade-aware: detect legacy `~/.claude/paul-framework/` installation, offer cleanup, and handle the upgrade path without data loss.

## Purpose
Users migrating from the old standalone PAUL framework have stale files at `~/.claude/paul-framework/`. The installer should detect this and offer to clean it up, providing a smooth upgrade experience consistent with PALS's "ultimate user friendly" value.

## Output
- Updated `install.sh` with legacy detection and cleanup
- Updated `uninstall.sh` with legacy cleanup option
</objective>

## AC-1: Legacy Detection

## AC-2: Legacy Cleanup on Install
```gherkin
Given a system with ~/.claude/paul-framework/ present
When install.sh runs
Then it offers to remove the legacy directory
And if accepted, removes ~/.claude/paul-framework/ completely

```

## AC-3: Clean Install Unaffected
```gherkin

When install.sh runs
Then it completes normally with no legacy-related output

```

## AC-4: Uninstaller Handles Legacy
```gherkin
Given a system that may have ~/.claude/paul-framework/ from a pre-PALS install
When uninstall.sh runs
Then it also removes ~/.claude/paul-framework/ if present
And reports legacy cleanup if performed
```

<tasks>

<task type="auto">
  <name>Task 1: Add legacy detection and cleanup to install.sh</name>
  <files>install.sh</files>
  <action>
    Add a new step between the repo validation (step 0) and the framework file install (step 1):

    1. Check if ~/.claude/paul-framework/ exists
    2. If found:
       - Print a clear message: "Legacy PAUL framework detected at ~/.claude/paul-framework/"
       - Print what will happen: "This will be removed (PALS replaces it at ~/.pals/)"
       - Prompt user for confirmation (y/N) — default No for safety
       - If confirmed: rm -rf ~/.claude/paul-framework/ and report success
       - If declined: print warning that legacy files remain but continue install
    3. If not found: no output, proceed silently

    Keep the prompt interactive (read from /dev/tty for piped installs).
    Use the same echo style as existing installer output for consistency.
  </action>
  <verify>
    - Run install.sh on a system with legacy dir present → should prompt
    - Run install.sh on a system without legacy dir → should skip silently
    - Script still passes shellcheck or bash -n syntax check
  </verify>
  <done>AC-1, AC-2, AC-3 satisfied: Legacy detected, cleanup offered, clean installs unaffected</done>
</task>

<task type="auto">
  <name>Task 2: Add legacy cleanup to uninstall.sh</name>
  <files>uninstall.sh</files>
  <action>
    Add a step after removing ~/.pals/ (step 1) to also clean up legacy paths:

    1. Check if ~/.claude/paul-framework/ exists
    2. If found: rm -rf ~/.claude/paul-framework/ and report removal
    3. If not found: skip silently

    No prompt needed — uninstall is already a destructive action the user chose.
    Place this alongside the existing stale ~/.claude/commands/pals/ cleanup.
  </action>
  <verify>
    - Ensure uninstall.sh removes legacy dir when present
    - bash -n uninstall.sh passes syntax check
  </verify>
  <done>AC-4 satisfied: Uninstaller handles legacy cleanup</done>
</task>
