<objective>
## Goal
Create the v2.51 validation classification baseline contract that defines the taxonomy, current suite inventory, preservation rules, and Phase 219 hardening handoff for Pi and cross-harness validation checks.

## Purpose
PALS must classify validation checks by support purpose before any check is weakened, relocated, archived, deleted, or treated as legacy-only. This baseline preserves command-output validation truth while making Pi-supported runtime, shared invariant, frozen legacy parity, runtime lens, guided UI safety, helper delegation, GitHub Flow safety, and artifact/process evidence explicit for future validation-script and legacy-cleanup work.

## Output
A new `docs/PI-NATIVE-VALIDATION-CLASSIFICATION-CONTRACT.md` artifact that:
- cites Phase 200 validation/evidence strategy and v2.47-v2.50 closure evidence,
- names the validation class taxonomy and blocking posture,
- inventories current `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` category families,
- records current command-output baselines as Pi 200/200 and cross-harness 116/116 from Phase 217 closure,
- defines count-change reconciliation requirements and no-cleanup preservation rules,
- hands Phase 219 exact scope for validation-suite classification hardening.
</objective>
