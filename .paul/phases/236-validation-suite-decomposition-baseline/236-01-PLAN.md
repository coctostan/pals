<objective>
## Goal
Create the Phase 236 validation-suite decomposition baseline and make the first behavior-preserving extraction from duplicated validation-suite helper code while preserving current validation command truth and TAP pass counts.

## Purpose
Phase 234 identified the two validation suites as large, marker/prose-heavy maintainability hotspots. Phase 235 classified validation-suite decomposition as risky work: it can reduce maintenance friction only if it preserves validation truth, command-output authority, module blockers, GitHub Flow gates, parent-owned APPLY, and explicit scope proof. This plan establishes the baseline map and performs one narrow helper extraction so future validation work can build from a safer structure instead of another prose-only inventory.

## Output
- New `docs/PALS-VALIDATION-SUITE-DECOMPOSITION-BASELINE.md` documenting current suite concerns, risk-tier expectations, category inventory, wording-sensitive checks, and future structured-assertion candidates.
- New `tests/helpers/tap.sh` containing shared TAP/reporting helper functions currently duplicated in the two validation suites.
- Updated `tests/pi-end-to-end-validation.sh` and `tests/cross-harness-validation.sh` sourcing the shared helper and carrying concise decomposition-baseline comments.
- Current validation counts preserved: Pi validation remains `209/209`; cross-harness validation remains `123/123`.
</objective>
