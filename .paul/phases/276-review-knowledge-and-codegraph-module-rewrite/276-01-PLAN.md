## Goal
Audit and rewrite the Phase 276 advisory module cohort — IRIS, REV, SKIP, CODI, DOCS, and RUBY manifest hook/on-demand descriptions plus their primary operational references — so review, knowledge, codegraph, docs, and debt guidance remains useful, compact, bounded, durable where needed, and non-blocking.

## Purpose
Phase 276 applies the v2.63 collaborative instruction-audit pattern to advisory and on-demand modules: audit first, then walk files one by one with visible before/after proposed edits before any mutation. These surfaces should improve PLAN/APPLY/UNIFY evidence without claiming lifecycle authority, blocking progression, reading broad context by default, inventing source facts, or persisting bulky/ambiguous artifacts. Phase 275 remains useful guidance, but Phase 276 must explicitly mirror v2.63's user-facing edit packets: current excerpt, proposed replacement, user approval, edit, and re-read.

## Output
A bounded advisory-module rewrite with:
- compact audit of IRIS, REV, SKIP, CODI, DOCS, and RUBY approved files plus relevant deferred principle files;
- blocking user approval checkpoint before source edits;
- approved file-by-file, section-by-section collaboration loop: within each file, handle one section/edit at a time by showing the current excerpt (`BEFORE`), concrete problem, proposed replacement/focused diff (`AFTER`), affected advisory/persistence semantics, verification expectation, user approval or wording change, then edit only that approved section;
- re-read verification after each approved section edit before moving to the next section in the same file or the next file;
- final validation evidence showing advisory modules do not block, persistent advisory artifacts have explicit destinations/schemas, broad or magical inference is removed, lifecycle authority stays with PLAN/APPLY/UNIFY, and scope stayed within approved instruction files plus routine `.paul/*` lifecycle artifacts.
</objective>
