<objective>
## Goal
Benchmark PALS against 15 agentic coding frameworks to identify gaps, differentiators, and v1.0 feature priorities.

## Purpose
Before committing to v1.0 scope, understand the competitive landscape. Research findings will directly shape which features to build, skip, or deprioritize — preventing wasted effort on solved problems and surfacing unmet needs PALS should address.

## Output
- `.paul/phases/34-competitive-research/competitive-analysis.md` — comprehensive analysis document with framework profiles, gap analysis, differentiator map, and v1.0 recommendations
</objective>

```gherkin
Given framework profiles are complete
When gaps are identified
Then each gap includes: what competitors offer that PALS lacks, severity (critical/important/nice-to-have), and effort estimate (small/medium/large)
```

```gherkin
Given framework profiles are complete
```


    2. **Architecture** — how it structures projects (files, config, conventions)

    4. **Target user** — who it's built for
    5. **Strengths** — what it does best
    6. **Weaknesses** — where it falls short
    7. **Relevance to PALS** — what PALS can learn from it

<task type="auto">
  <name>Task 2: Gap analysis, differentiator map, and v1.0 recommendations</name>
  <files>.paul/phases/34-competitive-research/competitive-analysis.md</files>
  <action>
    Using the framework profiles from Task 1, produce three analysis sections:

    **Gap Analysis:**
    - Compare PALS feature set (from README.md) against all frameworks
    - Identify features competitors have that PALS lacks
    - Rate each gap: critical / important / nice-to-have
    - Estimate effort: small (1 phase) / medium (2-3 phases) / large (4+ phases)
    - Group gaps by category (e.g., IDE integration, multi-model support, UI/UX)

    **Differentiator Map:**
    - Identify what PALS has that no/few competitors offer
    - Document why each differentiator matters
    - Note how to strengthen or leverage each one
    - Identify differentiators at risk (competitors catching up)

    **V1.0 Recommendations:**
    - Prioritize gaps as must-have / should-have / could-have for v1.0
    - Map recommendations to potential phases (Phase 35+)
    - Factor in: effort, impact, alignment with "Linux of Harness Engineering" vision
    - Include "do NOT build" list — things competitors do that don't fit PALS philosophy
  </action>
  <verify>Gap analysis has severity + effort ratings; differentiator map has leverage notes; recommendations have priority + phase mapping</verify>
