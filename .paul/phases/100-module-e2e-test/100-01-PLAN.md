<context>
## Module Hook Map (what we need to trigger)

### pre-plan (6 modules)
- TODD: Scan planned files for test files/frameworks
- DEAN: Run dependency audit
- DOCS: Scan for stale/missing docs
- IRIS: Scan files for anti-pattern signatures
- RUBY: Check files for tech debt
- DAVE: Check for CI/CD config patterns

### post-plan (1 module)
- TODD: If TDD type, restructure tasks

### pre-apply (2 modules)
- TODD: Verify test files exist or are first
- WALT: Run test suite, record baseline

### post-task (1 module)
- TODD: Run tests after each task, block on regression

### post-apply (7 modules)
- TODD: Run full test suite, suggest refactor candidates
- WALT: Re-run tests+lint+typecheck, compare vs baseline
- DEAN: Re-run dep audit, compare vs pre-plan
- DOCS: Compare changed files vs doc catalog
- IRIS: Run 25 review patterns against changed files
- SKIP: Scan for new decisions, suggest knowledge entries
- DAVE: Validate CI config if modified

### post-unify (3 modules)
- WALT: Compute quality delta, append to quality-history.md
- SKIP: Extract decisions from SUMMARY.md
- RUBY: Scan for new debt patterns

## Strategy to Trigger All Modules
Tell Pi to plan Phase 3 of pi-verify with these specific elements:
1. Add a new dependency (e.g., chalk for colored output) → DEAN
2. Add/modify test files → TODD
3. Add a GitHub Actions CI workflow → DAVE
4. Update README.md → DOCS
5. The code itself has patterns for IRIS to review
6. Make explicit decisions in the process → SKIP
7. Any code changes → WALT baseline + comparison, RUBY debt scan
