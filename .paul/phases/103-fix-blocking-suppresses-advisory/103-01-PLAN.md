<objective>
## Goal
Change module dispatch from "stop on first block" to "collect all results, display all output, then enforce blocks." This ensures advisory modules (IRIS, RUBY, DOCS, SKIP) always produce output even when blocking modules (TODD, WALT) trigger.

## Purpose
Phase 102 proved that when WALT blocks at priority 100, all advisory modules at priority 200-300 are skipped. Users miss security warnings, debt reports, and doc drift alerts because of a lint error. Advisory output should always be visible.

## Output
- Updated dispatch logic in 4 workflow files
- Reinstall to deployed location
</objective>

<context>
## Current Behavior (broken)
```
For each registered module (sorted by priority):
  Run module
  If action: block → STOP, surface reason
  // Lower-priority modules never run
```

## Desired Behavior
```
Phase 1 — Collect all results:
  For each registered module (sorted by priority):
    Run module
    Collect result: { annotations, action, reason }
    // Do NOT stop on block — continue to next module

Phase 2 — Display all output:
  Output full dispatch log with ALL module results
  Show advisory annotations (IRIS, RUBY, DOCS, SKIP)

Phase 3 — Enforce blocks:
  If any module returned action: block:
    Surface blocking reason(s) with collected advisory context
    Offer fix/override/stop
```
</context>

<acceptance_criteria>

## AC-1: Advisory Output Visible Despite Block
```gherkin

```

## AC-2: Block Still Enforced
```gherkin

When all modules have been collected
Then the block is still enforced — APPLY does not silently continue
```

</acceptance_criteria>
