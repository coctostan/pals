<objective>
## Goal
Harden the repo-source Pi extension artifact-slice runtime so existing PALS context slices follow `docs/PI-NATIVE-ARTIFACT-SLICE-CONTRACT.md`: named slice labels, source citations, freshness markers, explicit bounds, full-read fallback language, and non-authority wording.

## Purpose
v2.47 must turn the Phase 202 contract baseline into supported Pi runtime behavior before broader workflow capsules, helper compression, validation classification, or legacy cleanup. This plan scopes the first runtime hardening slice to the existing Pi artifact-slice path only, preserving `.paul/*` authority, PLAN/APPLY/UNIFY ownership, module evidence, GitHub Flow gates, parent-owned APPLY, explicit approvals/checkpoints, and full authoritative reads for consequential work.

## Output
- `drivers/pi/extensions/pals-hooks.ts` emits contract-shaped artifact slices for the implemented inventory subset: `current-lifecycle-state`, `active-roadmap-phase`, and `approved-plan-task-packet` when a current plan path is available.
- `drivers/pi/extensions/README.md` and `drivers/pi/skill-map.md` document named slices as derived orientation aids, not lifecycle authority.
- `tests/pi-end-to-end-validation.sh` contains runtime-lens marker checks for slice schema, activation gating, fallback language, read-only behavior, no hidden persistence, and no authority replacement.
</objective>

```gherkin
Given slices are only orientation aids
```
