<task type="auto">
  <name>Task 1: Add quick mode detection and execution path</name>
  <files>kernel/workflows/init-project.md</files>
  <action>
    Add a new step `detect_quick_mode` immediately after `check_existing` (before `detect_existing_code`):

    ```
    <step name="detect_quick_mode" priority="after-check-existing">
    **Detect quick init mode.**

    Check if the user's init request contains "quick", "fast", "--quick", or "just init":
    - Store `quick_mode = true` if detected
    - Store `quick_mode = false` otherwise

    **If quick_mode = true:**

    1. Infer project name:
       - From package.json `name` field if exists
       - Otherwise from current directory name
       - Store as `project_name`

    2. Ask ONE question:
       ```
       Quick init mode — one question only.

       What are you building? (1-2 sentences)
       ```
       Wait for response. Store as `description`.

    3. Apply all defaults:
       - `brownfield = false`
       - `planning_mode = "direct-requirements"`
       - `collaboration_level = "medium"`, `default_collaboration = "medium"`
       - `target_users_and_needs = "TBD — refine during planning"`
       - `problem_opportunity = description`, `why_now = "TBD"`
       - `must_have_scope = "TBD — define during first /paul:plan"`
       - `deferred_scope = "TBD"`, `out_of_scope = "TBD"`
       - `constraints = "None captured"`, `dependencies = "TBD"`
       - `assumptions = "TBD"`, `open_questions = "TBD"`
       - `success_signals = "TBD"`, `current_state_notes = "New project"`
       - Derive `core_value` from `description`
       - `module_selections` = all enabled
       - `integrations_enabled = false`
       - `specialized_flows_enabled = false`
       - `git_enabled = false` (detect git repo; if found set `git_enabled = true`, `git_workflow = "none"`)
       - All git automation = false

    4. Skip directly to `create_structure`, then `create_project_md`, `create_prd_md`, `create_roadmap_md`, `create_state_md`, create pals.json with defaults, then `confirm_and_route`.

    5. Skip: `detect_existing_code`, `determine_planning_posture`, `populate_from_codebase`, all `gather_*` steps, `configure_modules`, `configure_git`, `prompt_integrations`, `check_specialized_flows`.

    **If quick_mode = false:** proceed to `detect_existing_code` normally.
    </step>
    ```

    Also update the `confirm_and_route` step to show quick mode in the display when applicable:
    - Add after the Core value line: `Mode: Quick init — all defaults applied`

    Avoid: Changing any existing step logic — quick mode is a new branch that bypasses existing steps entirely.
  </action>

    - **Quick mode** (~1 question): User says "quick" → infer name, ask "what are you building?", default everything, create artifacts, route to /paul:plan
