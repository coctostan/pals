<objective>
## Goal
Extract the S6 `lifecycle-ui` subsystem from `drivers/pi/extensions/pals-hooks.ts` into a new sibling Pi extension module `drivers/pi/extensions/lifecycle-ui.ts` without changing visible Pi lifecycle status, lifecycle widget, loop-badge text, module-activity rendering, or `syncLifecycleUi` mutation behavior at any existing lifecycle event call site.

## Purpose
This continues the v2.60 sibling Pi extension extraction wave after Phase 257 established the S6 contract. The extraction reduces `pals-hooks.ts` responsibility while preserving exact Pi UI behavior at every existing `syncLifecycleUi(ctx)` call site, the runtime UI-element identifiers (`PALS_STATUS_ID = "pals-lifecycle"` and `PALS_WIDGET_ID = "pals-lifecycle"`), the loop-badge mark sequence (`PLAN✓ APPLY✓ UNIFY✓` formatted from the `[✓○]` regex), the action-label heuristics (`Complete` / `Blocked` / `Waiting` / `Ready` / `Next`), the module-activity rendering shape (`Modules: <stage> • <list>`), and the widget multi-line shape. It also preserves S4 canonical reply delivery, `.paul/*` artifact authority, transcript-visible canonical replies, no inferred merge intent, no auto-approval, no auto-continue, no skipped checkpoints, the **no-UI-only-lifecycle-decisions** invariant, and command-output validation truth.

S6 is the **first runtime-coupled sibling extraction** — it mutates Pi UI through `ctx?.ui?.setStatus` / `ctx?.ui?.setWidget` rather than purely producing strings. Phase 258 must therefore preserve every existing `syncLifecycleUi(ctx)` invocation in `pals-hooks.ts` by call shape; only the import source changes.

## Output
- New runtime sibling: `drivers/pi/extensions/lifecycle-ui.ts` exporting the eight S6 functions and defining the two S6 constants single-defined.
- Updated `drivers/pi/extensions/pals-hooks.ts`: imports the moved S6 functions/constants from `./lifecycle-ui`; keeps S4 canonical-reply delivery inline; keeps shared helpers and types defined; **newly exports** the four currently-inline symbols `MAX_VISIBLE_MODULES`, `RECENT_MODULE_ACTIVITY_LOOKBACK`, `parsePalsState`, `collectRecentAssistantTexts` so the new sibling can value-import them (Phase 254 precedent: Phase 254 exported the previously-inline `ActivationState` for the S7 sibling). Keeps every existing `syncLifecycleUi(ctx)` call site functionally identical.
- Updated `tests/pi-end-to-end-validation.sh`: one localized Pi-supported-runtime guardrail block proving the new sibling holds the two S6 constants + eight S6 functions + loader-compat marker + UI-mutation invocation rule, and that `pals-hooks.ts` no longer declares them inline while still keeping S4 inline and shared helpers/types intact.
</objective>
