<objective>
## Goal
Turn the completed audit, evidence-path fixes, and live Pi visibility work into a decision-usable recommendation set for the module system: explicit per-module keep/improve/merge/remove guidance plus bounded recommendations for proof, live visibility, validation, and end-report summaries.

## Purpose
Phase 68 proved the module system is more real than visible. Phase 69 hardened durable end-of-loop evidence. Phase 70 made recent module activity legible in Pi without creating adapter-owned truth. Phase 71 should now decide the long-term balance: what to keep as-is, what to improve, what to defer, and how `Module Execution Reports` and related proof/visibility surfaces should work without drifting into telemetry, Pi-only lifecycle state, or a second execution ledger.

## Output
- `.paul/phases/71-recommendations/module-recommendations-matrix.md` — per-module keep/improve/merge/remove recommendations with rationale, proof/visibility implications, and timing
- `.paul/phases/71-recommendations/proof-visibility-guidance.md` — bounded guidance for durable proof, live visibility, validation posture, and end-report/module-summary behavior
