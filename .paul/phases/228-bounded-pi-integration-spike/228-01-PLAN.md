# Phase 228 Plan 01: Artifact-Slice Targeting and Deduplication Runtime Spike

<objective>

## Goal

Implement the smallest safe runtime-lens spike for artifact-slice targeting and deduplication in the Pi extension. The spike must reduce duplicate lifecycle/roadmap/plan orientation in the bounded PALS context payload while preserving activation gating, source citations, freshness markers, output bounds, full-read fallback, and `Derived aid only` authority markers.

## Purpose

Phase 227 selected artifact-slice targeting and deduplication as the highest-value Phase 228 candidate because it can reduce repeated STATE/ROADMAP/PLAN orientation context without weakening `.paul/*` artifact authority, PLAN/APPLY/UNIFY lifecycle gates, module evidence, parent-owned APPLY, GitHub Flow safety, explicit approval/checkpoint behavior, or command-output truth.

## Output

- Runtime implementation in `drivers/pi/extensions/pals-hooks.ts` that narrows artifact-slice selection and trims duplicate orientation facts across emitted slices.
- Pi docs and skill-map updates describing the targeting/deduplication behavior and unchanged authority boundaries.
- Localized Pi validation guardrails in `tests/pi-end-to-end-validation.sh` proving runtime/docs markers and reconciling expected Pi validation count changes.
- No cross-harness validation script change unless APPLY discovers a shared invariant gap that requires re-planning.
