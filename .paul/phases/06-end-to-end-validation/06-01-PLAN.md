<acceptance_criteria>

## AC-1: Fresh Install Works from Clone
```gherkin
Given the PALS repo is cloned
When ./install.sh is run
Then all framework files exist at ~/.pals/
And all commands exist at ~/.claude/commands/pals/ and carl/
And hook is registered in settings.json
```

## AC-2: PALS Init Works on Test Project
```gherkin
Given PALS is installed
When /pals:init is run in a new project directory
Then .paul/ directory is created with STATE.md, PROJECT.md, ROADMAP.md
And the project is ready for /pals:plan
```

## AC-3: Full Loop Completes
```gherkin
Given a PALS project is initialized
When a plan-apply-unify loop is executed
Then PLAN.md is created
And APPLY executes tasks
And UNIFY creates SUMMARY.md
And STATE.md reflects loop completion
```

## AC-4: CARL Rules Activate
```gherkin
Given PALS is installed with CARL hook registered
When a new Claude Code session starts in a project with .carl/
Then CARL rules are injected into the session context
```
