<task type="auto">
  <name>Task 1: Investigate Claude Code context clearing mechanisms</name>
  <files>(no files modified — research only)</files>
  <action>
    Research the following approaches for programmatic context clearing:

    1. **Hook return signals**: Can a UserPromptSubmit hook return a signal that triggers /clear?
       - Review Claude Code hook documentation/behavior
       - Test if any hookSpecificOutput keys trigger actions

    2. **Slash command injection**: Can hooks inject slash commands into the conversation?
       - Review if additionalContext can contain executable commands

    3. **External process**: Can an external script monitor context and send commands?
       - Review if Claude Code exposes any IPC, socket, or CLI for sending commands

    4. **Instruction-based approach**: Can we inject strong instructions telling Claude to auto-pause?
       - The CARL hook already injects CRITICAL bracket warnings
       - Could this be enhanced to trigger /paul:pause automatically?
       - Key constraint: Claude can invoke skills, but /clear is a built-in, not a skill

    5. **Compact/optimize manual cycle**: If fully automatic isn't possible, how fast can we make the manual cycle?
       - Optimize handoff format for machine consumption
       - Reduce pause→clear→resume to minimal user interaction

    Document findings for each approach.
  </action>
  <verify>Research document created with findings for all 5 approaches</verify>
  <done>AC-1 satisfied: All mechanisms investigated and documented</done>
</task>
