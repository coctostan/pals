## API Reference
Pi extension API docs — ExtensionCommandContext section:
- `ctx.getContextUsage()` returns `{ tokens: number, ... }` (uses last assistant usage or estimates)
- `ctx.newSession({ setup: async (sm) => { sm.appendMessage(...) } })` returns `{ cancelled: boolean }`
- `pi.sendUserMessage(content)` triggers a new turn as if the user typed it
- `ctx.newSession()` is only available on `ExtensionCommandContext` (command handlers), not event handlers
</context>
