<objective>
## Goal
Implement the Phase 66 collaboration model so PALS planning workflows can run at low, medium, or high collaboration depth, support a project default plus per-run override, distinguish exploratory vs direct-requirements runs, and offer the 4-option pre-plan review menu before moving into execution.

## Purpose
Phase 65 established that PALS already has strong artifact authority and execution discipline, but its upstream planning conversations are not collaborative enough. This phase should improve interaction quality across shared planning workflows without weakening `.paul/*` lifecycle truth, backward compatibility, or cross-harness portability.

## Output
- Updated shared planning workflows and templates for collaboration-aware planning behavior
- Updated command and Pi skill guidance so wrapper entry points match the shared workflow semantics
- Updated Pi extension and validation coverage for the new review/routing behavior
</objective>
