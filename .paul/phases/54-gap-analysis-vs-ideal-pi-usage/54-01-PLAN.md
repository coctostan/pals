<objective>
## Goal
Define the ideal artifact-compatible Pi integration model for PALS, compare that target against the current implementation established in Phase 53, and produce a clear gap-analysis handoff that Phase 55 can use to generate recommendations and roadmap priorities.

## Purpose
Phase 54 turns the Phase 53 audit baseline into an actionable understanding of what is missing, weak, or underused in the Pi adapter. It matters because PALS must improve Pi-native fit without breaking its core value: shared markdown workflows remain authoritative across harnesses while drivers stay thin, modular, and portable.

## Output
- `.paul/phases/54-gap-analysis-vs-ideal-pi-usage/ideal-pi-integration-model.md` — explicit description of the ideal Pi integration target under artifact-compatibility constraints
- `.paul/phases/54-gap-analysis-vs-ideal-pi-usage/pi-gap-analysis.md` — dimension-by-dimension current-vs-ideal comparison with gap classification and evidence
- `.paul/phases/54-gap-analysis-vs-ideal-pi-usage/54-gap-analysis-report.md` — synthesized report separating critical gaps, secondary gaps, open questions, and Phase 55 recommendation inputs
</objective>
