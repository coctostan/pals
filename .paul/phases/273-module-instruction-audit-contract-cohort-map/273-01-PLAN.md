<objective>
## Goal
Create `docs/PALS-MODULE-INSTRUCTION-AUDIT-CONTRACT.md`, the v2.64 Phase 273 docs-only audit contract and cohort map that turns the v2.63 section-by-section workflow rewrite pattern into an executable module-instruction audit protocol for base and optional PALS modules.

## Purpose
v2.63 proved that instruction cleanup must be operational, semantic, and collaborative: add guardrails first, present a compact audit and proposed section order, wait for user approval, edit only the approved section, re-read the changed section, then validate. v2.64 applies that pattern beyond lifecycle workflows into module manifests, module references, module workflow overlays, and shared module-dispatch/base-module documentation. Phase 273 establishes the contract before any module source is rewritten so later phases do not devolve into bulk cleanup, marker-only compliance, or unbounded module churn.

## Output
A single docs-first contract artifact with:
- evidence base and source inventory for module instruction surfaces
- v2.64 audit rubric for context efficiency, semantic enforcement, lifecycle compatibility, hook behavior, and holistic correctness
- explicit holistic module-review questions for redundancy, confusion, stale ceremony, marker-only compliance, and simply wrong guidance
- cohort map for Phases 274–278 with module/file inclusion rules and boundaries
- collaborative rewrite protocol inherited from v2.63, including blocking approval checkpoints for later rewrite phases
- guardrail and validation handoff for Phase 278
- out-of-scope list and non-authority statement preserving `.paul/*` lifecycle truth
</objective>
