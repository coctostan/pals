    - How does the framework persist state across sessions?

    - Can it detect stale or outdated state?
    - Does it support multi-session or multi-agent collaboration?
    - How does it manage context window limitations?
    - What's the recovery mechanism if a session dies unexpectedly?

    **Domain 3 — Codebase Intelligence (brownfield equivalent):**
    - How does the framework understand an existing codebase?
    - Does it build reference graphs or dependency maps?
    - Does it detect coding conventions automatically?
    - Does it identify technical debt or concerns?
    - What's the adoption workflow for existing projects?
    - Does it support incremental/partial mapping?
    - How does it identify high-impact files (hubs, entry points)?

    - Per-framework findings (organized by domain)
