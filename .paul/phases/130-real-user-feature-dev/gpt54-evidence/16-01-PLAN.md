<objective>
## Goal
Add a `/verify diff` mode that scopes verification to files changed in git while preserving the existing `all`, `test`, `lint`, and `quick` flows.

## Purpose
This gives AI agents a genuinely useful incremental workflow: faster feedback on in-progress changes without forcing a full-project verification pass every time.

## Output
- New `diff` subcommand exposed in the Pi command and verify tool surfaces
- Verification logic that detects changed files from git and runs a scoped verification strategy
- Tests covering command parsing/help text and diff-specific verification behavior
</objective>
