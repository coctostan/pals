## AC-1: Installer Deploys All Pi Artifacts
```gherkin
Given the PALS repo with the Pi driver
When install.sh runs with PALS_DRIVER=pi in a temp HOME
Then the skill directory (~/.pi/agent/skills/pals/) contains:
  - 4 kernel subdirs (workflows/, references/, templates/, rules/) with files
  - 11 skill directories each containing a SKILL.md
  - modules.yaml with installed module entries
And the extension directory (~/.pi/agent/extensions/) contains pals-hooks.ts
```

## AC-2: Skill Structural Validity
```gherkin
Given all 11 installed SKILL.md files
When each is checked for required structure
Then each has YAML frontmatter with name and description fields
And each contains at least one workflow reference (indicating it delegates to kernel workflows)
And all 11 expected skill names are present (paul-init, paul-plan, paul-apply, paul-unify, paul-resume, paul-status, paul-fix, paul-pause, paul-milestone, paul-discuss, paul-help)
```

## AC-3: Extension Structural Validity
```gherkin
Given the installed pals-hooks.ts extension
When checked for required patterns
Then it exports a default function
And it registers all 11 /paul-* commands via registerCommand
And it handles session_start and context events
```

## AC-4: Driver Manifest Accuracy
```gherkin
Given drivers/pi/driver.yaml
When checked against actual installer capabilities
Then all declared capabilities (workflow_invoke, hook_register, state_read, state_write, user_interact, command_register) are supported by the installed artifacts
And the config.framework_dir matches the installer target
```

  <name>Task 1: Create Pi end-to-end validation script</name>
  <files>tests/pi-end-to-end-validation.sh</files>
  <action>
    Create a bash validation script following the TAP format pattern from tests/cross-harness-validation.sh.

    **1. Installer Deployment Checks:**
    - Create a temp HOME dir (use mktemp -d, trap cleanup on EXIT)
    - Run the Pi installer: `PALS_ROOT="$REPO_ROOT" HOME="$TEMP_HOME" bash "$REPO_ROOT/drivers/pi/install.sh"`
    - Verify ~/.pi/agent/skills/pals/ exists with kernel subdirs
    - Count kernel files in each subdir (workflows, references, templates, rules)
    - Verify 11 skill directories exist under the skill dir
    - Verify each skill dir contains SKILL.md
    - Verify modules.yaml was generated
    - Verify ~/.pi/agent/extensions/pals-hooks.ts was copied

    **2. Skill Structural Checks:**
    - For each of the 11 expected skills: check SKILL.md exists
    - Verify each SKILL.md has frontmatter (starts with ---)
    - Verify each SKILL.md frontmatter contains `name:` and `description:` fields
    - Verify each SKILL.md references at least one workflow file

    **3. Extension Structural Checks:**
    - Verify pals-hooks.ts contains `export default function`
    - Verify it contains all 11 command names in registerCommand calls
    - Verify it contains `session_start` event handler
    - Verify it contains `context` event handler

    **4. Driver Manifest Checks:**
    - Parse driver.yaml for all 6 capability fields
    - Verify config.framework_dir matches installer target path pattern
    - Verify install.sh and uninstall.sh exist

    Script outputs TAP format. Exits 0 if all pass, 1 if any fail.

    Avoid: modifying existing files, requiring network, leaving temp dirs (trap cleanup).
