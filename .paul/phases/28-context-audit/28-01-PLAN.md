<objective>
## Goal
Measure the context weight of every module's reference files, kernel workflow hook dispatch sections, and overall system overhead. Identify bloat, redundancy, and optimization opportunities. Then trim and consolidate to establish lean per-module context budgets.

## Purpose
The module system loads reference files into Claude's context window during hook dispatch. Without measurement, we don't know if 8 modules x 2-3 references is sustainable or if it crowds out the user's actual work. This audit ensures the "Linux of Harness Engineering" stays minimal by design.

## Output
- CONTEXT-AUDIT.md with complete measurements and findings
- Optimized reference files (trimmed, consolidated, redundancy removed)
- Per-module context budget documented
</objective>

    - Top 10 heaviest files ranked
    - Redundancy flags (similar content across modules)
    - Overall system total
  </action>
  <verify>CONTEXT-AUDIT.md exists with all measurement tables populated, every reference file accounted for</verify>

<task type="auto">
  <name>Task 2: Analyze findings and optimize reference files</name>
  <files>All module reference files listed in files_modified</files>
  <action>
    Based on CONTEXT-AUDIT.md measurements:

    1. Identify optimization targets:
       - Files over 150 lines → trim to essential content
       - Duplicate patterns across modules (e.g., multiple "-principles.md" files with similar structure)
       - Verbose examples that could be shortened
       - Boilerplate that doesn't add value

    2. Optimize each flagged file:
       - Remove redundant explanations (trust Claude to infer from concise instructions)
       - Collapse verbose examples into compact ones
       - Eliminate repeated "purpose" sections that restate module.yaml descriptions
       - Keep: actionable instructions, detection heuristics, specific patterns
       - Remove: philosophical preamble, obvious guidelines, over-explained rationale

    3. Do NOT:
       - Delete any reference file entirely
       - Change file names or locations
       - Remove content that is genuinely unique and actionable
       - Break the hook dispatch contract (modules must still function)

    4. After optimization, re-measure and update CONTEXT-AUDIT.md with:
       - Before/after comparison per file
       - Before/after comparison per module
       - Total reduction percentage
       - Per-module context budget (recommended max lines)
  </action>
  <verify>
    - All reference files still exist at original paths
    - No reference file exceeds 150 lines
    - CONTEXT-AUDIT.md has before/after comparison showing reduction
    - Total reduction is at least 20%
  </verify>
  <done>AC-2 and AC-3 satisfied: bloat identified, references optimized, budgets documented</done>
</task>
