## Goal
Upgrade the CODI module from Phase 170's hello-world scaffolding to a live pre-plan advisory overlay that: (1) extracts explicit symbols from the current phase scope via strict textual rules (no magic inference); (2) calls the codegraph `impact({ symbols, changeType: "behavior_change" })` tool once with the extracted symbol array, wrapped with call-time graceful failure handling; (3) formats the returned blast radius as a markdown sub-block; (4) returns it as `context_inject.blast_radius` so the plan-phase workflow surfaces it inline in the generated PLAN.md's existing `<module_dispatch>` section. ## Purpose
Phase 171 is the first real test of CODI's value thesis: proving that structural blast-radius facts can be pre-computed at plan time and carried into the plan artifact so the APPLY phase stops having to "remember" to call codegraph for boundary and call-site truth. The source plan (`~/pi/workspace/thinkingSpace/plans/pals-codi-integration-plan.md`) recommends `impact` as the first concrete tool surface because it gives the clearest downstream-change signal with the smallest cognitive surface area. Phase 171 also validates the call-time failure-handling contract established by Phase 170 — the honest answer for tool availability is "try it and skip cleanly on failure," not "proxy-check filesystem markers." Phase 172 will add default-on registration and a light `analyze_scope` tweak; Phase 173 will run the real-world trial and produce the v0.2 / iterate / kill decision. This plan is the content phase: everything downstream depends on CODI actually injecting useful structural facts.
