<objective>
## Goal

Prototype the first Phase 195 Pi-native spike: bounded, read-only artifact-slice context loading for explicit PALS activations.

## Purpose

Phase 194 established that Pi runtime assistance may reduce repeated context cost only when the output remains derived, bounded, disposable, source-cited, activation-gated, and backed by full-read fallback rules. This plan turns that architecture into a small adapter-source prototype without changing lifecycle authority.

## Output

- `drivers/pi/extensions/pals-hooks.ts` gains a bounded artifact-slice helper used only by the existing explicit PALS activation injection path.
- `drivers/pi/extensions/README.md` documents the artifact-slice behavior, citation/freshness/fallback contract, and no-hidden-state guardrails.
- `drivers/pi/skill-map.md` records how the slice layer fits the Pi command/skill/workflow surfaces.
- `tests/pi-end-to-end-validation.sh` gains structural checks proving source citations, freshness markers, bounded output, activation gating, and fallback-to-full-read markers remain present.
</objective>

## AC-1: Artifact slices are activation-gated and source-cited

```gherkin
Given a PALS project and an explicit `/paul-*` or `/skill:paul-*` activation signal
When the Pi extension builds the bounded PALS context payload
Then the payload includes artifact-slice content only through that activation path and cites `.paul/STATE.md`, `.paul/ROADMAP.md`, and any included prior PLAN/SUMMARY/module-evidence source paths.
```

## AC-2: Slice output is bounded, fresh, and fallback-safe

```gherkin
Given artifact-slice context is included in the payload
When a user or model reads the injected context
Then each slice exposes a freshness marker or unavailable-source note, the total artifact-slice content is bounded by an explicit constant, and the payload states that full authoritative reads are required for edits, decisions, ambiguity, stale data, or contested facts.
```

## AC-3: Runtime assistance remains read-only and non-authoritative

```gherkin
Given the artifact-slice helper reads `.paul/*` files
When it constructs runtime context
Then it performs no `.paul/*` writes, creates no Pi-owned lifecycle state, does not alter PLAN/APPLY/UNIFY semantics, and preserves the existing context hook as support-only deduplication.
```
