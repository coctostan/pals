<objective>
## Goal
Replace the minimal 3-module `pals.yaml` with a complete JSON config system covering all 8 modules, upgrade `/paul:config` from SonarQube-only to full module management, and wire module selection into `/paul:init` so new projects configure their pals upfront.

## Purpose
Users currently have no way to enable/disable modules after install without hand-editing YAML. The config command only manages SonarQube. New projects via `/paul:init` don't ask about modules at all. This makes PALS feel incomplete — users should control which pals are active.

## Output
- `pals.json` — new JSON config file replacing `pals.yaml` (all 8 modules listed with enable/disable + descriptions)
- Updated `/paul:config` command — shows module status, allows toggling modules on/off, retains SonarQube integration
- Updated `/paul:init` workflow — adds module selection step during project initialization
- Updated installer — reads `pals.json` instead of `pals.yaml`
</objective>
