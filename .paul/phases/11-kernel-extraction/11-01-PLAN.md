<objective>
## Goal
Build a `migrate.sh` script that restructures the PALS repo from flat layout into kernel/modules directories, creates module.yaml manifests, and updates all live references (symlinks, installer, uninstaller) atomically. Then run it.

## Purpose
Phase 11's goal is physically separating kernel from modules. The previous attempt (raw file moves) broke live infrastructure. This plan builds migration tooling that handles the move AND all reference updates in one shot, with validation before and after.

## Output
- `migrate.sh` — idempotent migration script with pre-flight checks and rollback on failure
- Repo restructured: `kernel/` + `modules/` directories with all files in correct locations
- `install.sh` updated to reference new paths
- `uninstall.sh` updated to reference new paths
- 3 `module.yaml` manifests created (TODD, WALT, CARL)
- All live references (symlinks, `~/.claude/hooks/`, `~/.pals/`) re-pointed
</objective>
