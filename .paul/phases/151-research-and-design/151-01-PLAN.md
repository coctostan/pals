<objective>
## Goal
Design the REV module: review prompt structure, module interface, pals.json config schema, subagent dispatch pattern, `/paul:review` UX, and PR-level hook integration. Produce a design doc that Phase 152 can implement directly.

## Purpose
REV is a new kind of PALS module — it dispatches work to an external subagent (pi-subagents) rather than running inline grep/eslint commands. The design needs to nail the prompt structure (what makes a great review), the dispatch mechanics (how PALS talks to the Agent tool), and the UX (how users trigger and consume reviews). Getting this right in research avoids rework in implementation.

## Output
`.paul/phases/151-research-and-design/rev-design.md` — comprehensive design doc covering all decision points.
</objective>
