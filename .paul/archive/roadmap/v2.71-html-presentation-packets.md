### Key decisions
- Contract-first: fix packet authority, storage, types, audience modes, static-HTML rules, and non-goals before any generation wiring (Phase 299).
- HTML presentation packets are derived, static, review-only artifacts; `.paul/*`, PLAN/APPLY/UNIFY, module reports, git/PR/CI state, and parent-run command output remain authoritative.
- Packet instances live under `.paul/presentation-packets/{phase-or-milestone}/{packet-id}.html`, excluded from hot-artifact byte budgets, regenerable and discardable without lifecycle-state changes.
- Audience modes (`quick brief`, `reviewer brief`, `full evidence brief`) alter density only, never authority or citation requirements.
- All four packet steps are optional, non-blocking, default-skip, main-session-only, source-cited, escaped, and incapable of approving/blocking/merging/rewriting lifecycle state (modeled on the v2.70 strategic-assessment precedent).
- Packets stay static and dependency-free: no JavaScript, network assets, external CSS/fonts/images, browser automation, hidden state, or telemetry — avoiding any revival of live Plannotator-style UI risk.
- Generation-only wiring; no `.html` instances committed. The derived/static guarantee is enforced with a `git ls-files` emptiness check (committed/hot-path state), not a filesystem scan.
- Validation guardrail literals match each installed file's actual casing/wording so parity checks pass deterministically and catch real drift.
